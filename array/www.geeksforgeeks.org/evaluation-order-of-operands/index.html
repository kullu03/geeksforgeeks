Evaluation order of operands

Consider the below C/C++ program. 

#include<stdio.h>
int x = 0;

int f1()
{
  x = 5;
  return x;
}

int f2()
{
  x = 10;
  return x;
}

int main()
{
  int p = f1() + f2();
  printf("%d ", x);
  getchar();
  return 0;
}

What would the output of the above program – ‘5’ or ’10’?
The output is undefined as the order of evaluation of f1() + f2() is not mandated by standard. The compiler is free to first call either f1() or f2(). Only when equal level precedence operators appear in an expression, the associativity comes into picture. For example, f1() + f2() + f3() will be considered as (f1() + f2()) + f3(). But among first pair, which function (the operand) evaluated first is not defined by the standard.

      • pajju

        @Venki, you are right. This is a very nice question. (Also gave me more clarity about Sequence points).

        This is Undefined behavior(UB) and not Implementation defined.
        I’ll substantiate this more clearly.

        The exact problem here is because there is no sequence point in between the function calls.
        f1()+f2();

        Note and Remember
        1. ; is a sequence point.
        2. function call is a sequence point.
        3. ‘+’ is not a sequence point.
        4. In between the same sequence point, if the value of a variable is changed more than once, its called UB.
        Here, the same global x value is changed twice between semicolon ;

        Note: ; is a sequence point.
        i.e f1() + f2();
        Is like writing
        (x=5)+(x=10)

        Expressions like x++ + x++;
        where x value is changed twice between the same seq-point is an UB as well.

        This is UB. And Not Implementation defined.

        Implementation defined means, the C standard says; compiler can evaluate either call to f1() first or f2() in the expression f1() + f2();

        Note:
        associativity comes into picture when there are more than 1 operator of the same precedence in the expression.
        say in the expression,
        f1() + f2() + f3();
        this calls (f1() + f2())+ f3()
        i.e as ‘+’ is L – R associative
        But in, f1 + f2();
        here only one ‘+’ operator is there, so no associativity rules apply here.
        In such cases, any compiler is free to call either f1() or f2() first. Implementation Defined!

        Note:
        Consider an expression
        fun1()+fun2();
        here there is no guarantee that fun1() will called first.

        But, in
        fun1(), fun2();
        It is guaranteed that fun1 will called first.
        As , is a sequence point.

        The standard does not define (outside of “sequence points”) which sub-statements/expressions are evaluated first.

        • pajju

          The following operators and conditions create sequence points:

          1. && – The left-hand side is fully evaluated before the right-hand side.
          2. || – The left hand side is fully evaluated before the right-hand side.
          3. , – The comma operator when used in an expression. It is not a sequence point when used to separate function parameters, which means that function inputs are not evaluated in any specific order.
          4. Function call – All inputs are evaluated before entering the function.
          5. ? : – The expression before the question mark is fully evaluated before any of the conditionals.
          6. ; – The statement end is a sequence point.
          7. if, switch, for, while, and do while – The “conditional” portion is fully evaluated before the body.
