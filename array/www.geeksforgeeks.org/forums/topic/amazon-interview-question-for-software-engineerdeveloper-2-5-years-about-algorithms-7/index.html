struct BST{

struct BST* getNode(char start, char end)
    
{
    
struct BST *temp;
    
temp = (struct BST *)malloc(sizeof(struct BST));
    
temp->start = start;
    
temp->end = end;
    
temp->visited = false;
    
temp->lc = temp->rc =NULL;
    
return temp;
    
}
    
void createBST(struct BST **root, char start, char end)
    
{
    
struct BST *temp;
    
if(!(*root))
    
{
    
temp = getNode(start, end);
    
*root = temp;
    
}
    
else if( (*root)->start >= start)
    
createBST(&(*root)->lc, start, end);
    
else
    
createBST(&(*root)->rc, start, end);
    
}
    

    
int findChain(struct BST *root, char start, struct BST *root1)
    
{
    
if(!(root1))
    
return 0;

root1->visited = true;

return 1;

if (root1->start >= start)
    
return findChain(root, start, root1->lc);
    
else if (root1->start < start)
    
return findChain(root, start,root1->rc);
    

    
}

    Consider each string as an edge in a graph. start and end characters will represent the vertices (u, v). Now this problem simply transforms into a problem of checking if a the graph is a cycle or not.

    So the list of start vertices (u) for all the edges = list of end vertices (v) for all the edges. (basic graph theory)

    Now simply create a hash table and insert the starting characters  and count for all strings.  For each end character check if it is present in has table. If all characters are present, then there is a cycle in graph else no cycle.

    Hi guys,

    It seems to me that it is a hamiltonian walk problem in a bipartite graph.

    Treating this problem  as a bipartite graph, it will have partition A and partition B.

    In the partition A insert the first letter of each word and partition B the last letter of each word.

    Lets suppose we have the words: {tuvwa, mnopq, abcde, efghi, ijklm, qrsct} as mentioned before.

    So, partition A will be { t, m , a, e, i, q} and partition B = {a , q, e, i, m, t}

    Now, we have to create an undirected edge from partition A to B for each node in A that have the same letter in B. I mean,

    A = { t, m , a, e, i, q}

    B = {a , q, e, i, m, t}

    we should have then a edge from t in partition A to t in partition B, m in partition A to m in partition B and so on. When we finish, all nodes in A must have an edge to a node in B, otherwise, there is no solution for the problem (there is no possible chain).

    If all nodes in A have an edge to a node in B we now have to check if there is a hamiltonian walk. If there is, this is the solution, otherwise, there is no possible solution.

    there is a cute little trick to do this problem ,

    first extract all the start and the end characters of the strings – > linear time

    then form a graph as follows ,

    start in any String x if x has start and end characters as a and b then add and edge between a and b in the graph labeling it with the string number of string x , do this for all the strings -> linear time

    find an eulerian path / circuit in the graph that will be the chain – > linear time ( Fluery’s algorithm )
    if no path/ circuit then report as chain can’t be found .
