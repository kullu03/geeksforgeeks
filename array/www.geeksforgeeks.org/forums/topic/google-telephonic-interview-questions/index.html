    [JAVA SOLUTION for #1]

    Implement a comparator to compare two characters based on a dictionary (let’s say MyDictionary) that gives the rank (or some other info about the relative order of characters) for a given character

    public class MyComparator implements Comparator<Character> {

        MyDictionary d = new MyDictionary();

        public int compare (Character c1, Character c2) {

            if (d.getRank(c1) > d.getRank(c2) ) {

                return 1;

            } else if (d.getRank(c1) < d.getRank(c2)) {

                return -1;

            } else {

                 return 0;

            }

        }

    }

    then using Collections.sort() , sort the characters by passing an instance of the above comparator

    Collections.sort (Arrays.asList(characters), new MyComparator<Character>()) 

    where ‘characters’ is the given input array

    For “Sort the input character array based on the dictionary given” all of the above solutions are O(n log n). It could be done in linear time:

    Assume characters are all ASCII, s1 – string to be sorted, s2 – character ordering string (for instance ABCDE…YZ is natural lexicographical ordering). H – hash table of size 256, characters hash to themselves, hash table contains mapping character->count, i.e. how many times given character has been seen.

    1) Go through s1 and put characters from it into hash table H (of size 256, each character hashes to itself). if there is already a character in H increase the mapped count. For “banana” you would get b=1 a=3 n=2 in hash table and zeroes for other missing characters. – O(n)

    2) Now go through s2 and for each character C from s2 which is in hash table H (i.e. H[C]>0) output it and H(C) times and set H[C] = 0. O(m+n), m = strlen(s1), n = strlen(s2). => O(m+n)

    3) For the remaining characters in H, where H[C]>0 output them. You can start traversing H from index (hash) 1 to size(H)-1, in this case it will produce natural lexicographical ordering. Alternatively traverse s1 and for each C in s1 where H[C]>0 output it H[C] times and set H[C]=0. it looks like counting sort. O(m), m=strlen(s1)

    Overall complexity: O(m+n), i.e. linear. Space complexity O(1).

    `In the 2nd question, we are essentially given “number of inversions” for every element in the unsorted list. e.g. consider the heights’ list [3 8 1 1 4 2]. Now we get the array [0 0 2 2 1 3] each element indicating the number of taller people on the left (there is a one to one mapping). So we have an array of pairs [(3 0) (8 0) (1 2) (1 2) (4 1) (2 3)]

    O(n^2) approach is to use the “reversed insertion sorting”

    1. Sort the pairs’ array in the usual lexicographical way. (actually we are sorting the persons in the increasing height)
    = [(1 2) (1 2) (2 3) (3 0) (4 1) (8 0)]

    2. Now we process every pair staring with the last to the first: we shift it to right until we cross the number of tall people equal to the second element of that pair.

    Initially = [(1 2) (1 2) (2 3) (3 0) (4 1) (8 0)]
    After processing 1st pair = [(1 2) (1 2) (2 3) (3 0) (4 1) (8 0)]
    After processing 2nd pair = [(1 2) (1 2) (2 3) (3 0) (8 0) (4 1)]
    After processing 3rd pair = [(1 2) (1 2) (2 3) (3 0) (8 0) (4 1)]
    After processing 4th pair = [(1 2) (1 2) (3 0) (8 0) (4 1) (2 3)]
    After processing 5th pair = [(1 2) (3 0) (8 0) (1 2) (4 1) (2 3)]
    After processing 6th pair = [(3 0) (8 0) (1 2) (1 2) (4 1) (2 3)]

    [3 8 1 1 4 2] thus gives us the original permutation.

    As stated in one of the earlier responses it is possible to do it in n.log(n) time using a balanced tree if there are no people of the same height standing next to each other in the original order. In that case just collapse them to 1 representative person and “expand” later on appropriately.

    See http://www.spoj.com/problems/ORDERS/ Note that in this problem heights are unique.
