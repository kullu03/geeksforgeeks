    It seems this is not just a simple problem. I tried to calculate all three values together – xor, sum and product – and by doing random tests I easily found a counterexample when all of these there values were equal but the arrays were different. And it’s not very difficult to prove that there may be such collisions.

    We can reformulate this problem to something like “build a hash function for a multiset of numbers” (despite an order of elements in the array). This is usually called “fingerprinting”. Also, we should think about some randomized algorithm because every hash function in a finite field may have collisions.

    I found an article and a paper about such an algorithm – http://tcs.nju.edu.cn/wiki/index.php/Randomized_Algorithms_%28Spring_2010%29/Fingerprinting#Fingerprinting_multisets
