    Most of the answers here are either too complicated or are not O(N) running time and O(1) space complexities. Anything that shifts the elements to the right, uses memmove, or has a nested while loop inside a for loop will be O(N^2).

    The way to do this is to first loop, count the negative values, then loop again and swap positive values with the next available open positive position as you iterate through the values. Code as follows:

    void SpecialSort(int* arr, int n)
{
    int negCount = 0;
    for (int i =0; i < n; ++i) {
        if (arr[i] < 0)
            negCount++;
    }

    int negIndex = 0;
    int posIndex = negCount;

    while (posIndex < n && negIndex < negCount) {
        if (arr[negIndex] < 0)
            negIndex++;
        else
            swap(arr[negIndex], arr[posIndex++]);
    }
}

    The answers posted thus far don’t solve the problem in O(N) running time and O(1) space. Anything that has to shift elements to the right, or use memmove in a loop, or have a nested while loop inside a for loop will be O(N^2), and anything using another array will have O(N) space.

    The solution is to count the number of negative integers, thus giving the first position that a positive integer will appear, and then start with the first element and swap it with the next available positive integer position until it is a negative number, and then move forward.

    code:

    
void SpecialSort(int* arr, int n)
{
    int negCount = 0;
    for (int i =0; i < n; ++i) {
        if (arr[i] < 0)
            negCount++;
    }

    int negIndex = 0;
    int posIndex = negCount;

    while (posIndex < n && negIndex < negCount) {
        if (arr[negIndex] < 0)
            negIndex++;
        else {
            int temp = arr[negIndex];
            arr[negIndex] = arr[posIndex];
            arr[posIndex++] = temp;
        }
    }
}

    Hi guys,

    So far there was no correct solution that satisfy three conditions 1. O(n) time, 2. O(1) time and 3. the relative order is maintained.

    To understand the problem, you should know that O(n) time for sorting problems is not possible using comparison sort algorithms (any algorithm that compares its elements). The best case scenario for comparison sorts is O(nlogn).

    However, O(n) time for sorting is possible using Counting sort. This will solve condition (1)

    For condition (2) O(1) space which is just constant space. If the problem asks for integers only. We can make two constant arrays of MAX_INT elements to count both negative and positive elements. This will satisfy condition (2) and allows our Counting sort algorithm to sort the array.

    Condition (3) is quite tricky for counting sort. I haven’t had a solution for this since counting sort doesn’t retain the information of the array (which allows it to sort in O(n) time).

    int* NegSort(int *v,int n) {
        int lastneg_i = -1;

        //int v[] = {2,3,-1,-3,-2,0,23,-6,-8};
        for ( int i = 0; i < n && n > 0; i++ ) {
                // push each positive element in front 
                for ( ; v[i] > 0 && i < n && n > 0; ) {
                        int tmp;
                        tmp = v[n-1];
                        v[n-1] = v[i];
                        v[i] = tmp;
                        // that's the trick, for each inner loop iteraction, decrease n, 
                        // so +1 iteraction in inner loop means -1 iteraction in outer one
                        n--; 
                }

        }

        return v;
}

int* NegSort(int *v,int n) {

        for ( int i = 0; i < n && n > 0; i++ ) {
                // search for a positive number
                for ( ; v[i] > 0 && i < n && n > 0; ) {
                        // push back the positive
                        int tmp;
                        tmp = v[n-1];
                        v[n-1] = v[i];
                        v[i] = tmp;
                        // now we known that on tail there is a positive number
                        // we decrease n ( array size ) so for each iteraction 
                        // in inner loop there is one less interaction in outer
                        n--;
                }

        }

        return v;
}

    Consider the example,
    -1 1 3 -2 2
    1. Parse the array once and find out the first occurrence of a positive element (A[1]=1). Call this position say ‘m’
    If A[i]>0:
    m=i;
    break;
    2. Parse the array second time from m+1 to n-1.
    3. If A[i] < 0 :
    INSERT A[i] in mth position and shift other elements (Linked list)
    m++;
    4. Else continue

    Time complexity : O(n)

    While the above 2 answers look better in terms of time complexity, the result is wrong. This is what you get with those algorithms:

    for the input array:
    -2, 4, 5, -4, 8, 7, -3, -5, 6, -1, 10, -10

    the output is:
    -2, -3, -5, -4, -1, -10, 4, 5, 8, 6, 7, 10

    but should be:
    -2, -4, -3, -5, -1, -10, 4, 5, 8, 7, 6, 10

    This question is too hard for an interview as far as I am concerned. If the constraint of space O(1) is changed to O(n) then it is a fair question. Trying to do it in place may be doable but is not something that can be figured out in a pressure situation of an interview.

    Here is a space O(n) solution: 

    
void nagatives_at_front(int array[], int count)
{
    int negCount = 0;
    for (int index = 0; index < count; index ++)
    {
       negCount += array[index] < 0;
    }
    
    size_t size = sizeof(int) * count;
    int *tmpArray = (int *)malloc(size);
    
    memcpy(tmpArray, array, size);
    
    int negIndex = 0;
    int posIndex = negCount;
    
    for (int index = 0; index < count; ++index)
    {
        int val = tmpArray[index];
        if (val < 0)
           array[negIndex++] = val;
        else
           array[posIndex++] = val;
    }
    free(tmpArray);
}
