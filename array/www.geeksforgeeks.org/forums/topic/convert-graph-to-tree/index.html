    find the median on the tree.

    1. take any node and find furthest node from it. Lets name the furthest node as n1.

    2. Take N1 and now find the furthest node from it. Lets name the new furthest node as n2.

    3. N2-N1 are two most furthest nodes in the graph.

    4. find the median of the path between N2 and N1 and take this median as root and form the tree.

    I may have an efficient algorithm. It is based on the fact that any connected acyclic graph is a tree, and any node could be the root. The problem is to find the root with the shortest depth.

    1. Choose an arbitrary node v with degree d where d > 1.

    2. Consider v to be the root of d subtrees.

    3. Traverse each subtree with your favorite DFS, keeping track of the maximum depth of each, and a leaf node at that depth. Also build a list of child->parent relationships. When you’re done you will know the d farthest leaves from v and know the path from each directly back to v.

    4. Choose the two farthest nodes v1 and v2 with depth d1 and d2 respectively, where d1 > d2.

    5. Follow the path from v1 back to v (using the child-parent list) and stop after traversing (d1+d2)/2 nodes. The node you stop at is the root of your tree. There may be more than one possible root node, and the one you find depends on the node p you start with.

    All the steps above take O(|V|) or O(1). This algorithm works with every graph I’ve tried.

    If you want to visualize the tree structure, convert it to a directed graph in O(|E|) time, and topologically sort it in O(|V|+|E|) time.
