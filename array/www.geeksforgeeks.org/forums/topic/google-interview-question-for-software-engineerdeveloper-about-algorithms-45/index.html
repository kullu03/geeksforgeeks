    Hi

    Sorry everyone.. there’s slight problem in my last solution… it wud generate different numbers with different probability….

    One call of RAND5 can produce 6 outputs… so 2 calls will produce 36 combinations and hence they can’t be divided into 8 equal sets… but 3 calls to RAND5 will produce 216 combinations which can be divided into 8 sets of 27 combinations each….

    How to do that:

    Call RAND5 three times… Let the outputs be n1, n2, n3…

    Since RAND5 generates outputs from 0 to 5, we can basically construct base 6 numbers from it… Consider n1 to be the number at units place, n2 to be at tens place, n3 to at hundreds place…

    Thus the number generated from 3 calls is ( in base 10 ), num = n1 + n2 * 6 + n3 * 36

    This will produce numbers from 0 to 215… Now we just divide the numbers in 8 sets and return accordingly

    Here is the code

    int RAND7()
    
{
    
int n1, n2, n3;
    

    
n1 = RAND5();
    
n2 = RAND5();
    
n3 = RAND5();
    

    
int num = n1 + n2 * 6 + n3 * 36;
    

    
if (num>=27*0 && num<=27*1-1)
    
return 0;
    
else if (num>=27*1 && num<=27*2-1)
    
return 1;
    
else if (num>=27*2 && num<=27*3-1)
    
return 2;
    
else if (num>=27*3 && num<=27*4-1)
    
return 3;
    
else if (num>=27*4 && num<=27*5-1)
    
return 4;
    
else if (num>=27*5 && num<=27*6-1)
    
return 5;
    
else if (num>=27*6 && num<=27*7-1)
    
return 6;
    
else if (num>=27*7 && num<=27*8-1)
    
return 7;
    
}

    Rand7() generates numbers between 0-7, and all numbers should have the same possibility (1/8).
    Say, Rand7() generates a number ‘n’. The possibility of 0< =n<6 is 3/4. The possibility of n=6 is 1/8. And also the possibility of n=7 is 1/8.

    int Rand7()
    {
    int n1 = Rand5();
    int n2 = Rand5();
    int nt = n1 + n2;
    //nt is between 0-10, but the possibility of every number is not same.
    //For example the possibility of nt=0 is 1/36. The possibilities are 0=>1/36 , 1=>2/36 , 2=>3/36, 3=>4/36 , 4=>5/36 , 5=>6/36 , 6=>5/36 , 7=>4/36 , 8=>3/36 , 9=>2/36 , 10=>1/36

    //The possibility of nt IN (6,7,8) is 9/36 (1/4), Not IN (6,7,8) is 3/4
    //you can choose another three numbers that the total possibility is 1/4

    if(nt == 6 || nt == 7 || nt==8)
    {
    if(n1>n2 || (n1==3 && n2==3)) return 6; //the posibility is 1/8
    else return 7; //the possibility is 1/8
    }

    return Rand5(); //the possibilty is 3/4
    }

    Better solution
    int Rand7()
    {
    int n1 = Rand5();
    if(n1 != 0) //if n1 == 0, nt can not be more than 5
    {
    int n2 = Rand5();
    int nt = n1 + n2;

    if(nt == 6 || nt == 7 || nt==8)
    {
    if(n1>n2 || (n1==3 && n2==3)) return 6;
    else return 7;
    }
    }
    return Rand5();
    }

    Worst case, Rand7() calls the function Rand5() 3 times.
    Best case, Rand5() is called twice.
    Average, Rand5() is called 2,5833 times
