    First sort the array in nlogn. Then for each element,find a pair(b,c) such that b+c=-element.

    Time complexity: O(N^2)

    Space complexity: O(1).

    But this will definitely not work if ordering of the elements matter. e.g. in place of {0 5 -5}, are we allowed to print -5,0,5?

    Below code doesn’t care for the ordering rather it prints all the triplets irrespective of their ordering.

    // a[] is sorted
    
void printTriplet(int* a,int size)
    
{
    
int i,low,high,temp;
    

    
for(i=0;i<size;i++)
    
{
    
low=i+1;high=size-1;
    
temp=-1*a;
    
while(low<high)
    
{
    
if(temp==a[low]+a[high])
    
{
    
printf("%d %d %dn",a    ,a[low],a[high]);
    
low++;high--;
    
}
    
else if(temp<a[low]+a[high])
    
high--;
    
else
    
low++;
    
}
    
}
    
}
