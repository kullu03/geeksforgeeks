    Given an array of size 100. The elements of the array are not sorted. What is the average number of comparisons needed in a sequential search to determine that the element being searched is not there?

  • May 4, 2010 at 11:18 AM #92076

    Anonymous

    I guess it would be (n+1)/2. Logic behind is the possible number of comparisons for various cases are 1,2,3…100, 100 being the worst case. Summing them it comes to n(n+1)/2n= (n+1)/2 which can also be represented as O(n) in asymptotic ananlysis.
