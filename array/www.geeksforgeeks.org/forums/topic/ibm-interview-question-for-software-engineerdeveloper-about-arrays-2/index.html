    Maximum Subset Sum problems are very popular with the Interview folks, I belive. I was asked one too for Amazon.

    Its a pretty simple logic for seemingly a tough question that at first look requires O(n**3) solution.

    Of course, there’s a better solution in O(n), where n is no. of elements in the array.

    (Note: I’m only extending Kadane’s algorithm for MaxSubsetSum to MaxSubsetProduct, nothing magical)

    Lets get to solving the problem at hand. From the property of multiplication, we do know that if when any +ve quantity greater than 0 is multiplied by anything <0 results in a sub-zero product.

    With that in mind, and the fact that there are no zeros and negative numbers in the array, the solution seems pretty trivial:

    0. Initialize max_overall = 1, max_local = 1, start = 1, start_index=end_index=1

    1. for i=0 to n

    1a. max_local = max (1, max_local * a)

    1b. if ( max_local == 1 ) start = i;

    1c. max_overall = max (max_local, max_overall)

    1d. if ( max_overall == max_local) start_index = start, end_index = i

    2. for i = start_index to end_index

    2a. print a

    3. end

    (http://geeksforgeeks.org/?p=576 Kadane’s algorithm)

    The problem gets interesting with the introduction of negative numbers. Again, lets get back to the basic algebra. We know that the product of two negatives give a positive. Hence, as long as we find two negatives, our product is going to be positive. If not, the sub array with a max product will lie else where.

    I’ll leave this part as an exercise for you. If you can’t solve it despite trying, I’d be more than happy to post a solution.
