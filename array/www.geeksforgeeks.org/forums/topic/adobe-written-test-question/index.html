    When an unsigned number is given a negative value a warning is issued by compiler and negative value is converted to equivalent unsigned number.

    In statement
    unsigned int y = -12;

    The value assigned to y = UINT_MAX + y(negative value) +1

    C/C++ automatically promotes a signed number to an unsigned number when a signed number is compared with unsigned number.

    In statement:
    if ( x > y)

    value of x is automatically promoted to unsigned value.
    Unsigned value of x = UINT_MAX + x (negative value) +1

    since UINT_MAX -2 + 1 > UINT_MAX – 12 + 1
    It prints x is greater.

    In addition to Varun’s answer…

    Well on many architectures, a negative number is stored in a 2’s complement form.
    unsigned int assigned a negative value is not an exception, the only difference is in the way you access that memory.
    Try this:

    unsigned int y =-12;
printf("int y value is %d\n",y);
printf("unsigned int y value is %u\n",y);

int x = -2;
printf("int x value is %d\n",x);
printf("unsigned int x value is %u\n",x);

    OUTPUT:-

    int y value is -12
unsigned int y value is 4294967284
int x value is -2
unsigned int x value is 4294967294

    When you compare unsigned int with signed int without any explicit typecasting, compiler automatically read ‘x’ as unsigned int to compare it with unsigned int ‘y’ thus clearly the output is x is greater.

    Note: value of unsigned int y does not become 12.
