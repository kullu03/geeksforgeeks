    @deep: Memoization comes along with the constraint of space. In this method I declared two matrix to memoiz power and multiply method. So if you create mul[1000][1000], it would be able to take care of numbers < 1000. but since we are multiplying the powers it can easily cross this limit.

    For example:

    Call stack of pow(10,6) would look like:

    > pow(10,3) * pow(10,3)

    > (10 * pow(10,1) * pow(10,1)) * (10 * pow(10,1) * pow(10,1))

    > (10 * 10 * 10) * ( 10 * 10 * 10)

    > (10 * 100) * (10 * 100) — if you create multiply matrix of [101][101] – this will work

    > (1000) * (1000) — how about this – for this we would need [1001][1001]

    > (1,000,000)

    We have to be a bit careful if we use memoization here, because we are multiplying the powers.

    BTW, did you initialize the matrices to ZERO!?

    Input doesn’t have to be large, it’s enough that result (a^n) exceeds SIZE and you will access array out of bounds.

    imo, memoization doesn’t have much sense here…

    you can make multiply function using additions same as the power function using multiplications:

    int mul(int x, int y) {
    
int r = 1;
    
for (; y; y >>= 1) {
    
if (y & 1) r = r + x;
    
x = x + x;
    
}
    
return r;
    
}
    

    
int pow(int x, int y) {
    
int r = 1;
    
for (; y; y >>= 1) {
    
if (y & 1) r = mul(r, x);
    
x = mul(x, x);
    
}
    
return r;
    
}
