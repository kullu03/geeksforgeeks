    How about just doing inorder traversals of both the trees A and B? Then, you have basically two strings (lets assume for now that nodes contain single characters) and all you have to do is to check whether string for tree B is a substring of string for tree A. Substring detection can be done by using any of the standard efficient algorithms like the Knuth-Morris-Pratt algorithm.

    Time Complexity for this method = Time taken for inorder traversal + Substring detection = O(n) + theta(n) for KMP … n is number of nodes in the larger tree

    Space Complexity = O(n)

    Alone inorder traversal is not sufficient if tree is not BST.

    Store inorder traversal of tree1 and tree2, in1 and in2 respectively.
    store pre-order traversal of tree1 and tree2, pre1 and pre2 respectively.

    if in2 is a substring of in1 and pre2 is substring of pre1 then tree2 is subtree of tree1.
