    Cups are arranged in following format

                                  |_|
    
|_|    |_|
    
|_|   |_|   |_|
    
|_| |_|   |_|  |_|
    
|_| |_| |_|  |_|  |_|

    Each cup has capacity C and once a cup gets full, it drops half extra amount to left child and half extra amount to right child

    for Eg : let’ first cups get 2C amount of liquid then extra amount C(2C-C) will be divided equally to left and right child cup of next level

    i.e. C/2 to left child and C/2 to right child

    Write a function which takes input parameter as amount of liquid poured at top (L) and height of particular cup (h) index of that cup (i) and it should return amount of liquid absorbed in that cup.

    Here each cup in each level contains the same amount of liquid since left and right receive the 1/2 of extra amount. Hence index have no use.

    you can get the amount of liquid required to fill the cups till level L using sum of GP.

    total_fluid_toFill_level_L = (2^L -1)

    we can determine if the cups are completely filled at level “h”

    if not, we have two options.

    1. Cups at level “h-1″ are completely filled

    — in this case calculate the extra fluid being poured at level h and divide it by 2*h to get the ans.

    2. Cups at level “h-1″ are not completely filled

    — in this case the cup in the question is empty as there is no extra liquid for level h.

    Iterate the structure level wise.

    Initially, Lets say at level 1, cup 1 we put L litre of liquid and each cup has capacity c. Iterate at level one, so now new value in cup 1 at level 1 will be c and increment the liquid in the child cups at level 2 i.e. cup 1 and cup 2 at level 2 by half of remaining liquid (L-C)/2 .

    Now iterate through level 2, if for any cup, you find the store liquid more than C, remove the extra liquid and put half of the extra liquids to the adjacent cup in lower level i.e.level 3………..

    Now, making a tree and do the procedure looks good, but then there will be time wastage in searching. We can implement this algo with arrays as follow:

    Note:

    at level 1, there is 1 cup (represented by (1,1) ) ->total 1 cup

    at level 2, there is 2 cups (cup (2,1) and (2,2) ) -> total 2 cups

    at level 3, there is 3 cups (cup (3,1) , cup (3,2) and (3,3) ) ->total 3 cups

    For the first cup at level x, there will be 1+ 2+3+4+…+x-1 previous cups .

    So if we store the data in an array, the index of the cup (x,y) will be x(x-1)/2 + y

    Also Note that children cups of cup (2,2) are cups (3,2) (3,3)

    Similarly children cups of cup (x,y) are cups (x+1,y)(x+1,y+1)

    Keeping these in mind…. here is my code in c++, 

    #include <iostream>
    
using namespace std;
    

    
int cal_index(int lev,int j)
    
{
    
return ((lev*(lev-1))/2)+j -1;
    
}
    

    
float fill(float L,float cap,int i, int h,int ht) //ht=total number of levels
    
{
    
int total_cups=cal_index(ht,ht)+1;
    
float* arr=new float[total_cups];
    
for (int a=0;a<total_cups;a++)
    
{arr[a]=0;}
    
arr[0]=L;
    
for (int level=1;level<=ht;level++)
    
{
    
for (int j=1;j<=level;j++)
    
{
    
int index,left,right;
    
index=cal_index(level,j);
    
if (arr[index]>cap)
    
{
    
float diff=arr[index]-cap;
    
arr[index]=cap;
    
left=cal_index(level+1,j);
    
right=cal_index(level+1,j)+1;
    
if(left<total_cups){arr

    
=arr

    
+diff/2;}
    
if(right<total_cups){arr

    
=arr

    
+diff/2;}
    
}
    

    
}
    
}
    
cout<<"---Printing the array---n";
    
for (int a=0;a<total_cups;a++)
    
{cout<<arr[a]<<"  ";}
    
cout<<"

    
n";
    
int x=cal_index(i,h);
    
return arr[x];
    
}
    

    
int main()
    
{
    
cout<<fill(6,1,2,2,4);// L=6 litre, capacity of each glass is 1, total height is 4
    

    
}
