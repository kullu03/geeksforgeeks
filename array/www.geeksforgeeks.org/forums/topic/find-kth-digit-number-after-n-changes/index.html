    There is a changing rule that, 0 -> 01, 1 -> 10. For example, after the change, 10 is 1001.

    Assume that the input is 0, after n changes of such rule, what is the Kth digit?

    I can only come with the brutal solution, which is as followings. However I believe there exist better solutions, please tell me~

    public char findKthDigit (int n, int k) {
    String str = “0”;
    for (int i = 0; i < n; i++) {
    StringBuilder sb = new StringBuilder();
    for (int j = 0; j < str.length; j++) {
    if (str.charAt(j) == ‘0’) {
    sb.append(“01″);
    } else {
    sb.append(“10″);
    }
    }
    str = sb.toString();
    }
    return str.charAt(k);
    }

    Hi! Look at the pattern of some first changes:

    i=0: 0
    i=1: 01
    i=2: 0110
    i=3: 01101001
    i=4: 0110100110010110
    i=5: 01101001100101101001011001101001

    As you can see, every time the number of digits doubles. Moreover, the first half of the new string is equal to the whole previous string; the second half is a mirrored previous string if the current change number is even, and is an inverted (negated) mirrored previous string if the current change number is odd. If you write this down, you should definetely see this pattern.

    Thus, we can solve this problem in O(N): first, calculate at which half K-th char is located in the final string (after N changes). This gives us whether we should consider the previous string normally or mirrored. Also, look at the parity of N – that tells us whether we should consider the previous string normally or inverted (negated). Perform this operation N times by going backwards from N to 1 and shrinking the length of the string every time twice. Finally, when you have come to the initial state 01, you can answer the question.

    By the way, what are the constraints on N and K? The length of the string number i is 2i-1, so if N > 64, the length of the string is greater than maximum value of int64 (unsigned long long).
    Also, the brute-force solution you have mentioned requires time and space proportional to the length of the final string which is O(2^n).
