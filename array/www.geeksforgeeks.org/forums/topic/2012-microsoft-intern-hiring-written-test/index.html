D?

ret = a;
    
return ;

    ( seems to be the correct answer.
    

    

    Dynamically allocated memory is allocated on stack and stays there even after the function returns.  It is not deallocated until explicitely deallocated using delete().
    

    

    Array is allocated on function call stack and the base address becomes invalid once the functions returns.  See following for details.

    Option (C) is the only correct answer.
    

    

    The problem with (A) is that even though you allocate memory using new, but you return a _copy_ of the value pointed to by a. You don’t return the pointer itself, which means memory leak. The caller would never be able to deallocate the memory in allocated in the function.
    

    

    The problem with ( is that you’re returning an address to the local array which gets destroyed on returning from the function, which means the caller receives a pointer which points to an object which doesn’t exist anymore. Dereferencing such a pointer would invoke undefined behavior, the most dangerous thing in C++ (and C as well).
    

    

    The problem with (D) is same as (. It just doesn’t return, rather copy the address of local variable to the output argument. 
    

    

    Hope that helps.
