    I was thinking about this, I wrote code as well. It should work.

    This method will find you the two majorities in an array:

    void findMajorities(int a[], int n, int &m1, int &m2)
    
{
    
int count = 1; m1 = 0;
    

    
// loop through all the array elements
    
for(int i=0; i<n; i++)
    
{
    
// if a is same as the last calculated majority element, then increase the count
    
if(a     == a[m1]) count++;
    
// else decrease the count
    
else count--;
    

    
// if count becomes zero
    
if(count == 0)
    
{
    
// persist the current majority index to m2 and current index to m1
    
// and set count = 1
    
m2 = m1;
    
m1 = i;
    
count = 1;
    
}
    
}
    
}

    But after running this method, there might be possibility that majorities count is not > n/3. So we need to check that separately:

    void printActualMajorities(int *a, int n, int m1, int m2)
    
{
    
int countM1=0, countM2=0;
    

    
for(int i=0; i<n; i++)
    
{
    
if(a == a[m1]) countM1++;
    
else if(a     == a[m2]) countM2++;
    
}
    

    
if(countM1 > n/3) cout<<"Majority one: "<<a[m1]<<endl;
    
if(countM2 > n/3) cout<<"Majority two: "<<a[m2]<<endl;
    

    
if(countM1 <= n/3 && countM2 <= n/3) cout<<"No Majorities!" <<endl;
    
}

    Cheers! let me know if it has some glitch! Time complexity is O(n) only with no extra space.

    The algorithm proposed by Kevindra is called Moore’s voting algorithm which works for > N/2 case, not for N/3 case.

    For example, it doesn’t work in this case.

    [1, 1, 1, 2, 3, 4, 5, 6]

    By the time the iterator reaches 5, count reaches 0 and m1 and m2 are updated to 5 and 0 respectively.

    After the next iteration, count reaches 0 again and m1 and m2 are updated to 6 and 5 respectively, which means we’ve lost track of the element 1 now.

    Reference for Moore’s Voting Algorithm can be found here http://www.geeksforgeeks.org/majority-element/

    I think you can modify the voting algorithm. We can always eliminate one of the elements with > n/3 occurences as long as we also eliminate 2 others. We can maintain the two current most frequent elements(top two) with their adjusted frequencies. If the next element is one of the top two elements, we just increase the adjusted frequency. If the next element is a totally different one, we reduce both adjusted frequencies by 1 and ignore the new element. If any of the adjusted frequencies fall to 0, we remove it from top two and we push the next element at frequency 1 into the top two. Finally we will be left with top two elements. we will need to check again if the two are actually more than n/3 occurences.
