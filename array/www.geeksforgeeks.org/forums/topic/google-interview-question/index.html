    You are given a n*n matrix.Every cell of the matrix contain some value (positive).

    You are currently standing at (0,0) and you have to go to right most and botom most cell ((n-1)*(n-1))

    You can move either right or down.

    Q1. Find the most optimal path.

    Q2. Find the second most optimal path.

    Q3. Find the kth most optimal path.

    We can easily use DP to find atleast the most optimal path.

    Say the given array is arr[n][n], let A[n][n] be the matrix we make in our DP solution where A[j] is the sum of all values we encounter in the path when we travel from arr[0][0] to arr[j]. Note that the path chosen is such that this sum is minimum.

    So A[j] = arr[j] + max (A[j-1], A[i-1][j]), based on the fact that we can reach i,j only from a cell on its left or top .

    Also we will have to pre-fill A[0][j] and A[0]

    Keep a backtrack pointer pointing to the cell previous in the path.

    Not very sure about the 2nd most optimal or rather kth most optimal way!

    Just my thought on this problem:

    I’ll assume that by optimal path you mean the one with least accumulated value when traveling through all of its cells. If that’s the case, then you could use Dijkstra for calculating it which is O(n + nlgn) and uses O(n) of space for the priority queue.

    In order to get the 2nd or kth most optimal path, I think you need to be able to consider more paths at once and sort them to some degree, so this is my proposed approach:

    Suppose the input matrix is A with size nxn, then create a matrix B of nxn with a min-heap on each cell and follow these steps:

    – For all B[i, 0] insert A[i, 0] into the heap.
    – For all B[0, j] insert A[0, j] into the heap.
    – For every (i, j) from (1, 1) to (n – 1, n – 1):
    – Create two temporary arrays for each heap from B[i-1, j] and B[i, j – 1]
    and for each value add the one from the current cell, A[i, j].
    – Insert first k items (no preference) from both arrays into B[i, j] heap.
    – For every remaining item, if its value is larger than the minimum of B[i, j] heap,
    then pop the top item of the structure and insert it. Otherwise, ignore.
    – Return minimum value of B[n – 1, n – 1] heap as the kth largest path value.

    *Note: If you need to get the actual path then along with the cost you need to save a backtrack pointer in the heap.

    Time complexity: O(n^2*lgk)
    Space complexity: O(n^2*k)

    Just a little fix on my time complexity analysis, building a heap of k items from two arrays with k size each (worst-case) takes O(k) and inserting remaining k (worst-case) items in this heap takes O(k*lgk), so the actual complexity of the overall algorithm is: O(n^2(k + k*lgk)) = O(n^2*k*lgk).
