    It is also possible to call the same functions as.

    add(10, 10);

    add(10, 10);

    Now compiler will be in dilemma to resolve the function name. To avoid such bugs, it will not include return type in function signature. I am not sure of any other concrete reason.

    For example, making the call explicit should help compiler in resolving the function name. Even that is not allowed.

    For more information in C++, see functions with co-variant return types.
