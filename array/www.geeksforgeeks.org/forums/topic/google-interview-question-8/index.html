    Here is another approach,

    1. Assume there are N # of strings

    Design a simple hash function, like the one below,

    Code:
    unsigned SimpleHash(void *base, int byteCount) {
    unsigned char *p = (unsigned char *)base;
    unsigned hash = 0;
    for(int i = 0; i < byteCount; i++) {
    hash += p[i];
    }
    return hash;
    }

    1. Create an integer array A of size N.

    2. Hash on each string and store the value in A

    3. Pick the smallest string in the input array, say, it was located at string. From the hash array, collect all the string having same hash value, and on the way mark those collected hash values to zero.

    4. Repeat step 3 till all the entries in A are marked to zero.

    Above method is based on hash collision. The hash function should return same value for all those strings having same combination of characters. We should handle cases like when a string hash to zero.

    Another Approach:

    Create map where key is sorted signature of an anagram, and value is list or vector of all those strings with same signature.

    Once map is built, collect the strings in sorted form.

    It can be done “in-place” in two steps

    1). Sort the vector of strings “vector v” by using the predicate as
    sort(v.begin(), v.end(), pred);

    2). definition of pred is like :-

    bool pred(string s1, string s2)
    {
    sort(s1.begin(), s1.end());
    sort(s2.begin(), s2.end());

    return (s1 < s2);
    }

    In python this can be done by:

    sorted(word_list, key=lambda word: ”.join(sorted(word.replace(” “, “”).lower())))

    where we sort the original list based on their anagram string. The key is the word converted into alphabetically ordered characters with no space. Thus the word which are anagrams will have the same string and they would be placed together.
