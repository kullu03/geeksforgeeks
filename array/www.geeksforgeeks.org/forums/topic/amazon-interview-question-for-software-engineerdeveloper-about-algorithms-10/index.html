  • August 1, 2014 at 7:42 PM #130515

    I think this is similar to the Three sum problem where in an array we have to find and print the three numbers if they have sum equal to a given value or not, so in this case also we cant first stable sort the points based on x-coordinates then sort based on y-coordinates. Now for each pair find all other points which will fall in line with that pair and print them. This will have O(n^2) time complexity which happens to be the best possible for Three sum problem.
