    Given an array of integers A, give an algorithm to find the longest Arithmetic progression in it, i.e find a sequence i1 < i2 < … < ik, such that A[i1], A[i2], …, A[ik] forms an arithmetic progression, and k is the

    largest possible.

    The sequence S1, S2, …, Sk is called an arithmetic progression if Sj+1 – Sj is a constant.

    This question can be solved by DP. Here is the idea:

    1. use F(i) to track the length of the longest arithmetic progression subsequence ended with a

    2. use S(i) to track the corresponding constant

    3. F(i) = 1 + max(F(j)) where 0 < j < i and a – a[j] = S(j)

    Actually, after thinking about it a little more, I think my previous post doesn’t work. Here is another new idea. For each node a, keep a map between all the possible intervals between a[j] and a where 0 <= j < i, and the corresponding maximum subsequence length with each interval and end with a. For example: given an integer array of int[] a = {2, 3, 4, 6, 9}, the map for last element a[4] of value 9 would be as following:

    7 -> 1 (i.e. between 2 and 9)

    6 -> 1 (i.e. between 3 and 9)

    5 -> 1 (i.e. between 4 and 9)

    3 -> 2 (i.e. between 6 and 9)

    The length for the last map entry is 2 because we have 3, 6, 9

    The code in Java is as following:


    
int find(int[] a) {
    
/* trivial case */
    
if (a == null || a.length <=1) return 0;
    

    
/* build map for a[0] */
    
List<Map<Integer, Integer>> list = new ArrayList<Map<Integer, Ingeter>>();
    
for (int i = 1; i < a.length; i++) {
    
map.put(a - a[0], 0);
    
}
    
list.add(0, map);
    

    
/* build map for rest of array elements */
    
for (int i = 1; i < a.length; i++) {
    
Map<Integer, Integer> map_i = new Hashtable<Integer, Integer>();
    
for (int j = 0; j < i; j++) {
    
int interval = a     - a[j];
    
Map<Integer, Integer> map_j = list.get(j);
    
if (map_j.containsKey(interval)) {
    
int len = map_j.get(interval);
    
map_i.put(interval, len+1);
    
} else {
    
map_i.put(interval, 1);
    
}
    
}
    
list.add(i, map_i);
    
}
    

    
/* calculate the max length of arithmetic subsequence that ends with each element */
    
int[] max_len_list = new int[a.length];
    
for (int i = 1; i < a.length; i++) {
    
Map<Integer, Integer> map = list.get(i);
    
for (Integer key : map.keySet()) {
    
if (max_len_list     < map.get(key))
    
len_list     = map.get(key);
    
}
    
}
    

    
/* go through max_len_list and get the overall max length of arithmetic subsequence for the array */
    
int max = 0;
    
for (int i = 0; i < a.length; i++) {
    
if (max < max_len_list    )
    
max = max_len_list    ;
    
}
    

    
return max;
    
}

    Simplified Pseudo-code for DP solution:

    for i=0 to size-1
	for j=0 to i-1
		if (an AP exists such that diff = a[i]-a[j] and ends at j )
			use this AP and update it to end at i
		else
			save info about new AP having 2 elems, ending at a[i] and diff = a[i]-a[j]
ans = non-existant AP with length = -1
for each AP
	if(AP.length > ans.length)
		ans = AP
return ans

    If I got that right, no need for a DP solution with the whole solution space.
    Here is a O(n) c# implementation…

    [sourcecode language="C"]
    public static int FindLongestArithmeticProgression(int[] arr, out int s, out int e)
    {
    s = -1; e = -1;
    if (arr.Length == 0) return 0;

    if (arr.Length == 1) return 0;

    s = 0; e = 0;
    int curDiff = int.MinValue;
    int diff = curDiff;
    int tmp;

    int curLen = 0;
    int maxLen = 0;

    for (int i = 1; i < arr.Length; i++)
    {
    tmp = arr[i] – arr[i – 1];
    if (tmp != curDiff)
    {
    if (maxLen < curLen && curDiff > 0)
    {
    maxLen = curLen;
    e = i – 1;
    diff = curDiff;
    }
    curLen = 1;
    curDiff = tmp;
    }
    else
    {
    curLen++;
    }
    }

    s = e – maxLen;
    return diff;//output is valid if diff is > 0
    }
