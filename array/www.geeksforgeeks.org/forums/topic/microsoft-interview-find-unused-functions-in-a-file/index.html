    Given a file which has many functions(c file including main),

    write an algorithm with necessary data structures to list functions which are unused when the code is executed from main.

  • July 14, 2012 at 5:53 PM #112826

    kartik
    Moderator

    We can solve it using graphs. We can consider every function as a vertex (or Node) of graph.

    First scan the file and create a node for every function.

    Now trace function calls starting from main(), when a function is called, make an edge from caller to callee.

    Finally, find all nodes which are not reachable from node of function main().

    Create a map of all the functions in the file and initialize to 0

    maintain a Queue

    1. start from main, add all the functions to queue

    2. func = Que.dequeue

    map[func] = 1;

    for all function calls in that function, add them to queue

    3. Repeat step 2 while queue not empty

    4. check the map for those functions whose value is 0. these are the unused functions

    Let us build a Hash-Table which contains all the functions in the given .C-file .

    While executing the file remove all functions which will be executed.
    If a function is not found in HashTable , it means that function has already been executed at least once.

    Now, functions left in Hash-Table are those which aren’t executed at least once.

    time complexity =O(time to execute given .C-file)
    Space complexity of above algo is high

    Class Union-Find algo would do the job. Represent each function declaration/definition by a node (hash of function name can be used as key). Initially all nodes belong to different set. Start from *main* and for each function that is called (depth first/breadth first traversal can be done), do union of (main and this function).

    In the end, the methods which are not in the set of *main* are not called from anywhere. complexity (nlogn).
