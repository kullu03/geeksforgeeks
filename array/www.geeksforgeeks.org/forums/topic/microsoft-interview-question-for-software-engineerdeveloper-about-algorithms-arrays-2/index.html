    Given an array, sort the array in such a way that the first part of the sorted array contains odd numbers, and the second part contains even numbers. The first part has to be sorted in descending order and the second part in ascending order. Not extra array, no pre or post processing and it has to use a conventional sorting mechanism.

    Hello,
    1. After multiplying with -1
    2. find out the difference between largest odd number and smallest even number and add 1 to it.
    3. Subtract this difference from all the odd numbers.
    4. Apply any other sorting logic.
    5. Revert the pre processing done.

    Another way is
    1. Move all the odd elements to the beginning of array.
    2. Find out the position of last odd element and apply sorting till to that element.
    3. From next to that element again apply sorting logic.
    4. TC: O(nlogn) SC:O(1)

    use a bubble sort:

    when two even numbers, e1 and e2 are adjacent and e1 > e2, swap e1 and e2
    when two odd numbers o1 and o2 are adjacent and o1 < o2, swap o1 and o2
    when an even number e1 and odd number o1 are adjacent swap e1 and o1

    * numbers a and b adjacent means: b immediately after a.
