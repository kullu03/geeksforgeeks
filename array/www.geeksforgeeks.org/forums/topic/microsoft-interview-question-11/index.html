    @kafee786 I’m not sure how Knuth Morris-Pratt algorithm helps could you explain?

    There is a O(n^2) algorithm and the logics goes like this:

    Outer loop: Loop through each element and find the longest possible sequence starting at that location.

    Inner loop/sub method (Find longest sequence): Maintain a min and a max variable; and for each element you visit starting from the index, i, update the min and max w.r.t. char value, and add this element to a Set. The exit condition is when you reach the end of the sequence, your HashSet’s length is equal to max – min + 1. Or when the HashSet’s length did not change after you have added another item in the set, ie. duplicate has found.

    ======

    ANOTHER way that I had is (maybe faster):

    1) Assuming that there are only 26 characters case insensitive; Create an array of 26 that serve as occurrence array where the value of the array would be the index of occurrence. After generating this array, we would have something like,

    A=[-1, 7, 9, 8, -1, -1, 2, 5, 6, 1, -1 ... ]
    
//[ a, b, c, d,  e,  f, g, h, i, j,  k ... ]

    Noticeably, we can find out a few continuous ranges; these will represent the longest possible continuous sequence. Starting from the longest one (using the above sample): Create a set of pairs of the index of the character and the character itself, so we may have something like:

    [{2, g}, {5, h}, {6, j}, {1, k}]

    Now, sort this range by the index of the characters, we get

    [{1, k}, {2, g}, {5, h}, {6, j}]

    From inspecting the above range, we know that we should have a valid continuous sequence if there are no gaps in the indices. If we do have a gap, the longest sub-range whose difference of the max/min of the character value and the difference of the max/min indices should be equal to form the next valid sub-range.

    With this in mind, we can iterate through the above range and stop at a gap (maintaining a max/min char value along the way), if this sub-range qualifies, then remember it and continue on until we are finished with the entire range.

    Now we should have at least one valid continuous sequence with a length of L. Now repeat the above process of all of the possible sequences, whose length is greater than L, in the array A above. And always remember the longest one.

    Now, at the end, we should have a longest possible sequence.

    The time complexity for this should be O(N*log N), since it is bounded by the longest single pass — the sorting of a range.

    And we will use up to 3 times the length of string to store aux information.
