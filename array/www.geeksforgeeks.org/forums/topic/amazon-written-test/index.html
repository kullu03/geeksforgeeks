    Well this can be done in O(n) . Note the following example

    4 3 2 1

    How many swaps will it require? Ans is 3. Please make sure you understand that it is 3. Important point to note is the answer can never exceed n-1.

    For any given order, swap 2,3,4,…. n in order and you have the sorted array. (Note that this doesnt violate the lower bound of O(nlogn). These swaps are not the normal comparisons.)

    Now note that the lowest number you need to swap (say i) is the number such that there exists a number larger than i to the right of i. and if you need to swap i (which brings i to the right most position), you need to swap each of j >i . Thus if we find i, the ans is (n-i+1).

    How to find the i?

    Ans: with a single pass find the k such that we have such a situation

    XXX 1 XXX 2 XXX 3 … XX k XX.

    Then i = k+1;

    Thus the problem can be solved in O(n)
