    @Ankit Gupta,I think u are in right direction.

    It is a version of Huffman Coding problem where only the data of the root node needs to be outputted after the tree is formed.

    first maintain a min heap data structure taking length of strings into account.

    Delete two minimum values from the heap,create a new node containing summation of both values,insert this new node into minheap.

    Update the min heap data structure.

    Continue till minheap contains only one node.

    Output the value of this node.

    struct node* HoffmanCoding(struct node *st[NUM] )
    
{
    
struct node *x,*y,*new;
    
prepare_heap(st,NUM);
    
do
    
{
    

    
x=extract_min(st);
    
y=extract_min(st);
    

    
new=(struct node*)malloc(sizeof(struct node));
    
new->freq=x->freq + y->freq;
    
new->left=x;
    
new->right=y;
    
st[++n]=new;
    
prepare_heap(st,n+1);
    
}while(n>0);
    
return new;
    
}

    This can be done easily by Binary Search Tree just like Min Heap. Below are the steps.

    1. Take each string one by one and put their length in BST. It will take O(nlogn) time.
    2. Create a variable total_min_cost and initialize it to 0.
    3. Now remove first min value node and second min value node from the BST. Add the value of second min node in first min node.
    4. Put this first node again in BST. Every time cumulatively add cost value (first min + second min) in our total_min_cost variable
    5. iterate step 2 until only one node remains in the tree i.e only root node.
    6. At the end total_min_cost gives minimum concatenation cost.
