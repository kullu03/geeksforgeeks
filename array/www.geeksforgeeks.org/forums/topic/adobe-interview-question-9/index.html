    I was asked this question in my interview with ADOBE

    I have a list of N teams T1, T2, T3 … Tn. Each of these teams has played a match against every other team. I have a function displayResult(Team T1, Team T2), it returns the team which won the match between any two given teams T1 and T2.

    I have to write the teams in an order such the (n-1)th team (in the order) had lost to the nth team which in turn had lost to (n+1)th team..

    for example if in a particular order, teams appeared as T1, T2, T3, T4 … then team T1 had lost to T2, T2 had lost to T3, and T3 had lost to T4… It may be possible that T3 lost to T1 .. but that need not be taken into consideration while writing the order. Only the neighboring elements should be such that the element on the left has lost to the element on the right.

    How will you write the teams in this order? Write a code for it

    Make all the necessary assumptions you need to solve the problem.

    This is nothing but a directed graph given which we need to traverse such that all nodes gets visited

    let T1,T2,T3,T4 are the four teams then we will represent outward direction from Ti to Tj if Ti has lost to Tj.

    So lets represent if given T1 lost to (T2,T3,T4) T3 lost to T2 and T4 lost to T3 and T2 so T2 has not lost any.

    so we represent +1 for outward direction and -1 for inward

    we have

    T1 T2 T3 T4

    T1 0 +1 +1 +1

    T2 -1 0 -1 -1

    T3 -1 +1 0 -1

    T4 -1 +1 +1 0

    Just use directed graph traversal if all nodes visited we have the solution like we have here from T1 lost to T4 lost to T3 lost to T2 .

    There are 2 methods to do this:

    1st method: Similar to quick-sort. Consider there are n teams.Pick any of the team Ti and compare it with all the other (n-1) teams just like the quick sort algo. You will get two list seperated by Ti- one with which Ti lost and the other with which Ti won. Now apply the same algo on the two sublists till you have only one element in each list. This will be of complexity O(nlog n)

    2nd method: Similar to binary search tree. Pick any of the team Ti as the root of a BST. Keep inserting all the other teams to this tree following the principle that, if the team to be inserted has lost to the parent node (team) in the tree, then insert it to the left subtree, otherwise insert it to the right subtree. The inoeder traversal of this tree will give the desired output.
