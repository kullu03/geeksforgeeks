    Now consider matrix:

    ………..1……….. 5…………… 6………………. 7

    ………..2 ……….7……………. 8………………. 9

    ………..3………. 8……………. 9……………. 10

    Sorted sequence will be ->

    1 2 3 5 6 7 8 9 10

    Ok, now the idea is start two travellers starting at the arr[0][0]. Of course if k=1, arr[0][0] is ans.

    Now Row_Traveller travels across rows (downwords) and Column_Traveller travels across columns (rightwords). Whoever sees a smaller value travels one position ahead in its travelling direction.

    Meanwhile we keep a counter of total no of positions travelled by both. Once it reaches k we have our answer at the latest location skipped by either of traveller.

    Also, when row traveller reaches last row, it advances to next column and hence column traveller also advances. Be careful at such boundary cases.

    If both elements are equal both will advance and count increaments by 1 (If we want same elements to be counted only once) .

    Example:

    Col_Traveller —>

    Row_Traveller

    |

    V

    ………..1……….. 5…………… 6………………. 7

    ………..2 ……….7……………. 8………………. 9

    ………..3………. 8……………. 9……………. 10

    Let k = 6

    Then travelling is as follows:

    Row_Traveller…………….Col_Traveller………………..Count(Elements skipped)

    2,1………………………………….1,2…………………………………….1

    3,1………………………………….1,2…………………………………….2

    1,2………………………………….1,3…………………………………….3

    2,2………………………………….1,3…………………………………….4

    2,2………………………………….1,4…………………………………….5

    3,2………………………………….2,3…………………………………….6

    Since count reaches k, the latest element skipped that is arr[2][2] = 7 is the answer.

    Please note how we advance the Col_Traveller.

    I think complexity is O(M x N).

    Any suggestions or modifications?

    psychocoder
    Participant

    1.Allocate a new array,say temp of size N (assume the matrix is of order M X N).
    2.Store the elements of the last row into the matrix in reverse order.
    3.Treat the array temp as a max heap – no need to build it separately as the last row was sorted in ascending order.
    4.Extract the top most element from the heap and insert the next element of the same column(if there is no next element i.e. this element was at the 0th row then copy the last element from the temp array at the 0th position and decrease the size of the array by 1) .Then invoke the method maxheapify to reconstruct the heap.
    5.Repeat the above method for k times and the last element extracted from the heap was the kth max of the 2D array.
    Time Complexity: O(klogk)
    Auxiliary Space: O(k)

    start from 0,0 cell of the matrix and go along diagonally. At each step in diagonal direction we see we have crossed (i)^2 smallest elements on moving i steps diagonally. For eg. if we are at j th step then we know j^2 smallest elements and in previous step we had known (j-1)^2 smallest elements. now let (l-1)^2 < k <= l^2.
    l are integers because its the number of steps diagonally. now go along column up and also row right by comparing row elements with column element decrementing l^2 by one at each step until it becomes k. This gives the answer and took O(n) where n is max(number of rows, number of columns).

    Since the array rows and columns are sorted maximum element will be the a[n][n] and smallest element would be element at a[1][1].

    In first iteration

    Replace a[n][n] with a[1][1] and a[1][1] with some value less than it.

    start from a[n][n] and keep pushing it towards a[0][0] (by replacing it with largest of two adjacent cells) unless it is greater than both of its adjacent row cell and column cell.

    After k – 1 such iteration , a[n][n] will be kth largest element

    Psuedocode
    ==========
    `
    for(int p=1;p a[n][n]=a[1][1];
    a[1][1]--;
    for(int i=n,j=n;i>=0 && j>=0;) {
    int li=i;
    int lj=j;
    int largest = a[i][j];
    if(j>= 0 && a[i][j-1] > largest) {
    largest = a[i][j-1];
    lj=j-1;
    li=i;
    }
    if(i>=0 && a[i-1][j]>largest) {
    largest = a[i-1][j];
    li=i-1;
    lj=j;
    }

    if(a[i][j] == largest) {
    break;
    } else {
    //swap with largest
    a[li][lj] = temp;
    a[li][lj] = a[i][j];
    a[i][j] = temp;
    i=li;
    j=lj;
    }
    }
    }
