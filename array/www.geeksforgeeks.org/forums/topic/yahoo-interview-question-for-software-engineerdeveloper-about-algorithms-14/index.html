    this problem is different from the balanced partition problem because here it is given that the two sets have equal number of elements. there is no such constraint on the balanced partition problem.

    this problem can be solved by sorting the array in non increasing order. so array A = { 4,2,1,3,6,9}

    becomes = { 9,6,4,3,2,1}

    Start processing this array from the beginning and keep the sum of the elements in each of the sets upto that point. now consider next two elements at each step. assign the larger of these to the set with the smaller sum. update the sum and continue this procedure.

    so here

    9 | 9 3 | 9 3 1

    6 | 6 4 | 6 4 2

    I’ve a greedy approach…so it mayn’t be correct so plzzzz..verify..

    1. We need equal number of elements in both arrays.So it means at any time only even number of elements can be picked from the set.

    2.Now, sort the array.Traverse the array from left to right and store the minimum difference found b/w two consecutive elements.

    3. Assign 1st element out of the two to one set and the 2nd to the other set.

    4.Remove those two elements from the array or mark it as if there is no element at those indices.

    5.In the second iteration pick other two elements which are closest and assign the bigger element to the set which has lower sum and the smaller element to the other set.

    6.Keep on repeating these steps and store the minimum difference found so far.
