    Backtracking is overkill. @Ravi Teja’s method with binary searching can do the job. For more mathematical introduction on the problem, see the following link,

    http://en.wikipedia.org/wiki/Zeckendorf's_theorem

    Fibonacci series have numerous real world applications.

    Given below is the code for printing binary pattern of a natural number expressed in Fibonacci base,

    #include <iostream>

    using namespace std;

    #define MAX_TBL_SIZE (46)

    int floor(unsigned fibTable[], int l, int r, unsigned key)

    {

    // largest value <= key

    int mid;

    while( r - l > 1 ) {

    mid = l + (r - l)/2;

    (fibTable[mid] <= key ? l : r) = mid;

    }

    if( fibTable[r] <= key )

    return r;

    return l;

    }

    void constructTable(unsigned fibTable[], int size)

    {

    // Series considered

    // 1, 2, 3, 5, 8, 13, ...

    fibTable[0] = 1;

    fibTable[1] = 2;

    for( int i = 2; i < size; i++ )

    fibTable = fibTable + fibTable;

    }

    void makeMapping(unsigned fibTable[], char fibMarker[], int size, unsigned N)

    {

    // Add trivial case for zero

    unsigned n = N;

    int index = 0;

    while( n )

    {

    index = floor(fibTable, 0, size-1, n);

    fibMarker[index] = 1;

    n = n - fibTable[index];

    }

    }

    void printBaseFib(char fibMarker[], int size)

    {

    int i = size-1;

    while( i >= 0 && !fibMarker[i--] ) ;

    for( ++i; i >=0 ; i-- )

    cout << (int)fibMarker;

    }

    void testCase_01(void)

    {

    unsigned fibTable[MAX_TBL_SIZE];

    char fibMarker[MAX_TBL_SIZE] = {0};

    int size = MAX_TBL_SIZE;

    int N = 15;

    constructTable(fibTable, size);

    makeMapping(fibTable, fibMarker, size, N);

    printBaseFib(fibMarker, size);

    }

    @gogoplex Yes, there are multiple fabo representations possible for a number.

    For example: 16 can be represented as (13+3), and now we can break down 13 and 3 also to {8+5} and {2+1} so one more solution would be 8+5+2+1.

    we can break the numbers down until they become adjacent. In the above, you can’t break any number further because 8,5 are adjacent and 2,1 also.

    Ravi’s solution would give you the first solution i.e. 13,3. Because he using greedy approach here. However if you use DP you will get all the possible solutions, the time complexity would be 2^n though without memoization:

    int results[MAX]; // array to store the results
    
solve( int a, int b, int number, int i, boolean taken )
    
{
    
if( b > sum || sum < 0 ) return ;
    
if(sum == 0) { printArray(results);  return;}
    

    
if(taken) { results = a; } // store the number in results if taken
    

    
solve(b, a+b, sum-b, i+1, true); // take the next fabonacci
    
solve(b, a+b, sum, i, false); // ignore the next fabonacci
    
}

    The starting call for 15 would be solve(1,1, 15, -1, false);

    This code will print all the possible solutions.
