    To Satya:

    you can have a look the wiki link, where it explains the algorithm

    in detail.

    The basic idea is to split the array into two well balanced part, based

    on a carefully chosen pivot, in a similar way as quick sort. This reduces

    the original problem into a smaller size array. The same procedure is

    repeated recursively.

    Now, the question is how to pick a good pivot. For this part

    Selection Algorithm splits the array into n/5 parts, and computes

    te median of each part. The median of these n/5 value is computed

    again recursively using the selection algorithm, and this is used

    as pivot.

    Posting the code will be quite messy, but if you are interested,

    have a look at the wiki link, or Chapter 9 of Cormen.
