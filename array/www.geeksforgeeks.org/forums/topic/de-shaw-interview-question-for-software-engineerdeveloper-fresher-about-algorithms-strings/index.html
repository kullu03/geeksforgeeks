    I think what we should do is following :

    first check length of s3 is equal to the sum of lengths of s1 and s2.

    Take first and third sequence : find longest common subsequence of s1 and s3, if it is equal to s1, then go ahead. else return false.

    Now take second and third sequence : find longest common subsequence of s2 and s3, if it is equal to s2, then s3 is made up of s1 and s2.

    Since longest common subsequence problem can be done in O(n) time and O(n2) space, so run time complexity is O(n) and O(n2) space.

    Take 3 Indexes pointing to S1, S2, S3, now

    for each item in S3 Index,

    check if it is matching with S1 Index, if yes, increment S1, S3 index by 1,

    else check if it is matching with S2 Index, if yes, increment S2, S3 index by 1,

    else return false.

    boundary condition checks have not been mentioned. but always have to be checked for S1 or S2 boundary.
