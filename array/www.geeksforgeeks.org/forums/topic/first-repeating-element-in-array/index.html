    If the elements range is given, it is matter of using hash table to trace the duplicate. Otherwise, it not possible to detect duplicate in O(n) time (check element uniqueness on wiki). Here is pseudo code,

    scan array to get min and max values

    range width = max – min + 1

    build a hash table of size “range width”, reset all to zero

    for each X in Array

    index = A[X]-min

    if( HASH[index] not zero)

    duplicate found

    else

    increment HASH[index]

    @rukawa, thanks. @Guddu Sharma, we can keep an index as suggested by @rukawa, but in this case we

    need to scan entire array and update hash table. Also, update the index to keep track of first duplicate.

    [EDIT]

    nearest = ARRAY_END

    for each X in Array

    index = A[X]-min

    if( HASH[index] not sentinel )

    if(nearest > HASH[index] )

    nearest = HASH[index]

    else

    HASH[index] = index

    int FirstDuplicate(int A[], int size)

    {

    int min, max;

    int range;

    int index;

    int nearest = size;

    Range(A, size, min, max);

    range = max - min + 1;

    int *hash = new int[range];

    memset(hash, -1, sizeof(hash[0]) * range);

    for( int i = 0; i < size; i++ )

    {

    index = A - min;

    if( hash[index] != -1 )

    {

    if( hash[index] < nearest )

    nearest = hash[index];

    }

    else

    hash[index] = i;

    }

    delete[] hash;

    return nearest;

    }

    This solution is working fine for elements < 1000 in O(n)

    public class FirstRepeatingElement {

    public static void main(String[] args) {

    int[] n = { 3,2,2,4,4 };

    int len = n.length;

    int min = Integer.MAX_VALUE;

    boolean[] visited = new boolean[1000];

    for (int i = len – 1; i >= 0; i–) {

    if (visited[n]) {

    min = Math.min(min, i);

    } else {

    visited[n] = true;

    }

    }

    if (min == Integer.MAX_VALUE)

    System.out.println(“repeating element not found”);

    else

    System.out.println(“repeating element is ” + n[min]);

    }

    }
