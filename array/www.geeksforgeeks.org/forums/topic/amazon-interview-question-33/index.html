    You are given an array of Strings, return TRUE if and only if All the strings can be connected in one chain.

    Condition for connectivity is , if the last character of one string matches first character of second string, the two string can be connected.

    Example : String []arr ={"abc", "cde", "cad" , "def" , "eac"} will return TRUE coz all strings can be connected in one chain
    "abc"->"cde"->"eac"->"cad"->"def"

    Another Example : String []arr ={"acb" , "cde", "def", "ead" } returns False coz "cde"->"ead"->"def" is the possible chain but “acb” is left out.

    Note : Its not necessary to start with the first String and form the chain, It may happen that you won’t get a chain if you start with first string but you can get a chain if you Start with any other String. If there exists a possible chain, then your solution should return True.

    In the second example , if the first String was suppose “fcb” , then a possible chain would have exists "cde"->"ead"->"def"->"fcb" so True.

    Consider strings as nodes of directed graph, and add edges between strings where the last letter in one string is equal to the first letter in another (only in this direction!).
    Thus, the problem is reduced to finding whether Hamiltonian path exists in this graph.
    As known, this problem is NP-complete.

    The problem is not necessary NP hard because the graph built from the set of strings is not any graph but limited to a specific type of graph (see the first point below).

    I suggested to solve the problem as follows (please correct me if I am wrong):

    1. My first observation is that the graph can contain a directed loop, if and only if the all the nodes in the loop corresponding to strings having the same starting and ending character, e.g. aba, aca, aaa, aa, ama. These strings actually can be grouped together which form a clique in the graph. Therefore I suggest to group all of them together to form a single node, say a*a. We do the same thing for any group of strings that having the same starting and ending character. In this way we simplify the graph because the graph now contains no loop. This step can be easily done in linear time by using hash table to map character to strings with starting and ending with the corresponding character.

    2. When the graph is directed contains no loop (directed and acyclic ), we can solve the Hamilton path problem in linear time by first ordering the vertices by DAG topology sort. Then check if there is a Hamilton path as described here

    lam
    Member

    Thanks for correcting me.
    I think the following solution is close the the ground truth, though did not check carefully, please correct me again if I am wrong.

    I solve the problem with assumption that there is no string that starts and ends with the same character (no strings like a*a, b*b). The general case without this assumption can be solved by trivial extension of the following solution.

    In order to simplify the explanation I solve the following problem, check if there is a Hamilton cycle (instead of Hamilton path). The case with Hamilton path is discussed latter.

    For every character c, denote the set of strings that starts by c as S_c end the set of strings that ends with c is E_c. By the assumption that there is no string that starts and ends with the same character S_c and E_c must be disjoint and any string in S_c must connect only to all string in E_c.

    An important observation is that if there is a Hamilton cycle then |S_c|=|E_c| for any c. The question is that if this condition holds then there is a Hamilton cycle. Fortunately, the answer is yes with an additional condition: the graph must be connected. In fact if that condition holds, we can create a Hamilton cycle in linear time by starting from an arbitrary node and in every step choose an unvisited neighbor to move to. The latter operation is always possible because the graph is connected and there always a vacant node in E_c to move from a node from S_c.

    The condition for Hamilton path is:
    1. graph is connected
    2. Either: |S_c|=|E_c| for any c or this condition will hold when we are able to add one extra string such that the condition |S_c|=|E_c| for any c holds.

    How to handle the case when there are strings like a*a, b*b. It is easy, first remove all of them from the set of string. When a Hamilton path is found we just need to check for additional steps whether we can insert these strings at any point of the path, if yes then we done, if not then the answer is no hamilton path.
    I hope it is clear.

    I believe for this question it is more a Euler path problem rather than Hamiltonian path problem, while you could treat 26 letters as nodes and the String as edges. And for Euler path problem, there is classical algorithm (Of course I tried on my laptop and it worked for all above samples).

    1. Check whether the graph is connected.
    2. Check every node in the graph, whether the in degree equal to out degree and there at most one “-1″ degree and one “1” degree. Details may refer to Wiki.

    bool isChainExist(const char *p[], int size)
    {
    char a[size],b[size];
    for(int i=0;i {
    int lastel=strlen(p[i])-1;
    a[i]=p[i][0];
    b[i]=p[i][lastel];
    }

    int count=0, i=0,j=1, flag=1, last=0;
    while(count {
    while(j { flag=1;
    if(i!=j && a[j]==b[i])
    {
    a[j]=a[i]=’0′;
    i=j;
    j=0;
    count++;
    flag=0;
    break;
    }
    j++;
    }
    if(flag)
    {
    last++;
    i++;
    j=0;
    }

    if(flag && last==2)
    break;
    }

    return (count==size || count==size-1)?1:0;
    }
