    I guess you mean the “max” heap data structure.

    Better approach depends on objective and input data. If the objective is to get the third largest in minimum numbers of comparisons, tournament tree is best.

    If the data is duplicated, arrange the data in sets (bins – each bin will have signature which is element itself), and get max three elements from these sets. If the requirement is to get three largest distinct elements, pick first three bins.

    If the objective is to get these three elements in single pass as the data set arbitrarily long in size, it will be good approach to keep track of largest three elements in three different variables.

    int ThirdLargest(int A[], int size)
    
{
    
int max1, max2, max3;
    

    
max1 = max2 = max3 = A[0];
    
for( int i = 1; i < size; i++ )
    
{
    
if( A > max1 )
    
{
    
max3 = max2;
    
max2 = max1;
    
max1 = A    ;
    
}
    
else if( A     > max2 && A != max1 )
    
max2 = A    ;
    
else if( A     > max3 && A != max1 && A != max2 )
    
max3 = A    ;
    
}
    

    
return max3;
    
}

    //Finding third largest element of an array
    public static void findThirdLargest(int[] array) {
    if(array == null || array.length < 3) {
    System.out.println("No third largest element present");
    return;
    }

    TreeSet treeset = new TreeSet();
    for(int curr : array) {
    if(treeset.size() < 3)
    treeset.add(curr);
    else {
    int lowestElement = treeset.first();
    if(curr > lowestElement) {
    treeset.remove(lowestElement);
    treeset.add(curr);
    }
    }
    }

    System.out.println(“third lagest no is “+treeset.first());

    }
