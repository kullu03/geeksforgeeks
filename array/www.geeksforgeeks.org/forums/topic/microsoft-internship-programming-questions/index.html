    Find the majority duplicate element which occurs more than n/2 times in a given array of size n. Can this be done in O(n) time and O(1) space?

  • August 24, 2013 at 4:08 PM #122350

    if we know the range of elements , for eg. elements can be set of numbers ranging from 0 to 9, then make an int array of size range that we know with initial initialisation 0. this would take constant space = O(1)
    scan the given array of elements and increment the created array cell corresponding to that element. This would take O(n) complexity. Scan the created array once to find which cell have number greater than n/2.
    element corresponding to that cell is the answer.

    consider the first element as the majority element and initialize its frequency count as 1.
    Now starting from second element of the array, reduce the frequency by 1 if the element is not equal to majority element. If the freq becomes zero, choose next element as the majority element.
    Continue this until the second last element in the array.

    This will return the majority element only if it exists otherwise the result is unpredictable.

    As matter of some optimization, we can return if at any stage in loop, freq is greater then the remaining elements in the array.

    Pseudocode
    ==========

    int freq = 1;
    int majority = a[1];
    for(int i=2;i if(a[i] != majority)freq–;
    if(freq == 0) {
    majority = a[i+1];
    freq = 1;
    }
    }
    print(majority)
