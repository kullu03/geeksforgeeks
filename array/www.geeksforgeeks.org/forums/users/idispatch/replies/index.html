    For “Sort the input character array based on the dictionary given” all of the above solutions are O(n log n). It could be done in linear time:

    Assume characters are all ASCII, s1 – string to be sorted, s2 – character ordering string (for instance ABCDE…YZ is natural lexicographical ordering). H – hash table of size 256, characters hash to themselves, hash table contains mapping character->count, i.e. how many times given character has been seen.

    1) Go through s1 and put characters from it into hash table H (of size 256, each character hashes to itself). if there is already a character in H increase the mapped count. For “banana” you would get b=1 a=3 n=2 in hash table and zeroes for other missing characters. – O(n)

    2) Now go through s2 and for each character C from s2 which is in hash table H (i.e. H[C]>0) output it and H(C) times and set H[C] = 0. O(m+n), m = strlen(s1), n = strlen(s2). => O(m+n)

    3) For the remaining characters in H, where H[C]>0 output them. You can start traversing H from index (hash) 1 to size(H)-1, in this case it will produce natural lexicographical ordering. Alternatively traverse s1 and for each C in s1 where H[C]>0 output it H[C] times and set H[C]=0. it looks like counting sort. O(m), m=strlen(s1)

    Overall complexity: O(m+n), i.e. linear. Space complexity O(1).
