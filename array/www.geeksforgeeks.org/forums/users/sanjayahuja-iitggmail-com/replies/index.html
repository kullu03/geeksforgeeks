    consider the first element as the majority element and initialize its frequency count as 1.
    Now starting from second element of the array, reduce the frequency by 1 if the element is not equal to majority element. If the freq becomes zero, choose next element as the majority element.
    Continue this until the second last element in the array.

    This will return the majority element only if it exists otherwise the result is unpredictable.

    As matter of some optimization, we can return if at any stage in loop, freq is greater then the remaining elements in the array.

    Pseudocode
    ==========

    int freq = 1;
    int majority = a[1];
    for(int i=2;i if(a[i] != majority)freq–;
    if(freq == 0) {
    majority = a[i+1];
    freq = 1;
    }
    }
    print(majority)

    Here is recursive solution to do this
    class Node {
    public int data = 0;
    public Node next;

    public Node(int d) {
    data = d;
    next = null;
    }
    }
    public static Node swapAlternateNodes(Node head) {
    if (head == null)
    return null;
    if (head.next == null)
    return head;
    Node t = swapAlternateNodes(head.next.next);
    Node newHead = head.next;
    head.next.next = head;
    head.next = t;
    return newHead;
    }

    Since the array rows and columns are sorted maximum element will be the a[n][n] and smallest element would be element at a[1][1].

    In first iteration

    Replace a[n][n] with a[1][1] and a[1][1] with some value less than it.

    start from a[n][n] and keep pushing it towards a[0][0] (by replacing it with largest of two adjacent cells) unless it is greater than both of its adjacent row cell and column cell.

    After k – 1 such iteration , a[n][n] will be kth largest element

    Psuedocode
    ==========

    li=i;
