    printf("%cn",--as);

    gives garbage as it it trying to access an undefined memory location.

    printf("%d",&a[0]);

    gives the address of the first element of array a.

    printf("%c",*p);

    gives the first element of array a.

    *p++;

    Since unary * and ++ are right to left associative , it first increments p and then dereferences it.

    printf("%cn",*p);

    gives B since now p has been incremented in previous statement.

    printf("%c %c " ,*++p,*p);

    gives C because unary and ++ are right to left associative so p increments first and then dereferenced and it gives B because it is evaluated first in the printf arguments. You simply cannot predict it and the behaviour is usually unspecified.

    You may want to have a look at: http://stackoverflow.com/questions/621542/compilers-and-argument-order-of-evaluation-in-c/621548#621548

    Note:

    The statement:

    i = *(++p);

    Is the equivalent of:

    p += 1;

    i = *(p);

    The statement:

    i = *(p++);

    Is the equivalent of:

    i = *(p);

    p += 1;

    4 40 is correct because you are creating an array of pointers to integers in the statement

     int *p[10]

    Since the addresses in gcc compiler are of size 4 bytes, you’d get 4 when you print

    sizeof(*p)

    *p refers to the first element of the array which is a pointer to a integer.

    You get 40 as 4(size of a pointer)x10(no of elements) since sizeof(p) gives size of the array pointed to by p.
