    Here is a O(n) solution. This however assumes that you know the range of numbers coming in these pairs and that range is not too large.

    I use some storage for results(connectedNodes, validSets) which is needed but as per requirement the algorithm here is based on an array “links”. links basically represents the set to which the number i belongs.

    @srinivasareddyp how do you ensure that Step 1 is not of O(n) complexity? If Step 1 is of O(n) complexity then your algorithm will be of O(n^2) complexity.


    
package com.dsa;
    

    
import java.util.ArrayList;
    
import java.util.HashSet;
    
import java.util.List;
    
import java.util.Set;
    

    
public class App {
    

    
public List<Set<Integer>> findConnectedNodes(List<Pair<Integer>> pairs) {
    
List<Set<Integer>> connectedNodes = new ArrayList<Set<Integer>>();
    
List<Boolean> validSets = new ArrayList<Boolean>();
    
int[] links = new int[1000];
    
for (int i = 0; i < 1000; i++) {
    
links = -1;
    
}
    
for (Pair<Integer> pair : pairs) {
    
if (links[pair.first] == -1 && links[pair.second] == -1) { // new pair
    
Set<Integer> set = new HashSet<Integer>();
    
set.add(pair.first);
    
set.add(pair.second);
    
connectedNodes.add(set);
    
validSets.add(true);
    
links[pair.first] = connectedNodes.size() - 1;
    
links[pair.second] = links[pair.first];
    
}
    
if (links[pair.first] != -1 && links[pair.second] == -1) { // first number already seen
    
connectedNodes.get(links[pair.first]).add(pair.second);
    
links[pair.second] = links[pair.first];
    
}
    
if (links[pair.first] == -1 && links[pair.second] != -1) { // second number already seen
    
connectedNodes.get(links[pair.second]).add(pair.first);
    
links[pair.first] = links[pair.second];
    
}
    
if (links[pair.first] != -1 && links[pair.second] != -1 && links[pair.first] != links[pair.second]) {
    
/* linked pair but in different lists as of now */
    
if (connectedNodes.get(links[pair.first]).size() >= connectedNodes.get(links[pair.second]).size()) {
    
connectedNodes.get(links[pair.first]).add(pair.second);
    
validSets.set(links[pair.second], false);
    
} else {
    
connectedNodes.get(links[pair.second]).add(pair.first);
    
validSets.set(links[pair.first], false);
    
}
    
}
    
}
    
List<Set<Integer>> uniqueConnectedNodes = new ArrayList<Set<Integer>>();
    
int index = 0;
    
for (Boolean valid : validSets) {
    
if (valid)
    
uniqueConnectedNodes.add(connectedNodes.get(index));
    
index++;
    
}
    

    
return uniqueConnectedNodes;
    
}
    

    
class Pair<T> {
    
public final T first, second;
    

    
public Pair(T first, T second) {
    
this.first = first;
    
this.second = second;
    
}
    
}
    

    
public static void main(String[] args) {
    
App app = new App();
    
List<Pair<Integer>> pairs = new ArrayList<App.Pair<Integer>>();
    
// {1,5},{5,9},{7,8},{9,8},{3,12},{3,4}
    
pairs.add(app.new Pair(1, 5));
    
pairs.add(app.new Pair(5, 9));
    
pairs.add(app.new Pair(7, );
    
pairs.add(app.new Pair(9, );
    
pairs.add(app.new Pair(3, 12));
    
pairs.add(app.new Pair(3, 4));
    
List<Set<Integer>> connectedNodes = app.findConnectedNodes(pairs);
    
for (Set<Integer> nodes : connectedNodes) {
    
System.out.println(nodes);
    
}
    
}
    
}

    If you are allowed to use extra space, this can be done in linear time. Use a dictionary/HashMap, where keys are the numbers and values are counts.

    Using a Java syntax this would be: Map<Integer, Integer> dict;

    So you do something like this,


    
for(int num : numbers) {
    
if (dict.containsKey(num))
    
dict.put(num, dict.get(num)+1);
    
else
    
dict.put(num, 1);
    
}
    
int max = secmax = -1;
    
int maxFreqNum = secondMaxFreqNum = -1
    
for(Integer num : numbers) {
    
if (max < dict.get(num)) {
    
max = dict.get(num);
    
maxFreqNum = num;
    
secmax = max;
    
secondMaxFreqNum = maxFreqNum;
    
}
    
}

    The insertions take up a total of O(n) time and then second iteration again takes O(n) time in total because every iteration does O(1) operations.

    There is an alternative solution than sorting the strings/words in the dictionary provided you know that the alphabet from which these words are formed is finite and small. For example, English language letters, S={a,b,c,…,z} and size of S=26. The sorting method takes O(nlogn) time for insertion and O(1) time for retrieval.

    The dictionary can then be implemented as a hashtable. Using Java syntax:

    Map<Integer[], Set<String>>

    The keys of the map are integer arrays, which are nothing but an alphabet-wise count of the words in the dictionary. So, for each word you insert in the dictionary, do this: for each character of the word increment corresponding slot in the integer array. Anagrams will have identical integer arrays and hence same key.

    When you need to find a word calculate its integer array and retrieve the set of words from the hashtable. This method has complexity O(n) for insertion and O(n) for retrieval.

    I don’t program in C++ but I just looked up the C++ reference for std::rotate. If I am not wrong, std::rotate rotates the given list of elements such that the middle element comes to the first position.

    As per the question, you should use constant space. This can be done easily by maintaining two pointers. We have 2 cases.

    Case 1: number of elements is odd, mid element = (0+last_index)/2

    Let p point to 0th element, q to mid. Swap pth and qth element. Increment p and q. Keep on swapping till p equals mid index and q equals last. Now we just need to move the last element to correct place. To do this, set p = q-1 and start swap pth and qth elements. Keep decrementing p and q and swapping until p < mid index.

    Case 2: number of elements is even, mid element = (0+last_index)/2

    Let p point to 0th element, q to mid. Swap pth and qth element. Increment p and q. Keep on swapping till p equals mid index and q equals last.

public class ClosestPairFinder {
    
public Point[] findClosestPair(List<Point> points, List<Point> X, List<Point> Y) {
    
Point[] closestPoints = new Point[2];
    
if (points.size() > 3) {
    
// Divide part
    
int midX = (X.get(0) + X.get(X.size()-1))/2;
    
List<Point> pointsL = new ArrayList<Point>();
    
List<Point> pointsR = new ArrayList<Point>();
    
List<Point> xL = new ArrayList<Point>();
    
List<Point> xR = new ArrayList<Point>();
    
List<Point> yL = new ArrayList<Point>();
    
List<Point> yR = new ArrayList<Point>();
    
for(Point p : X) {
    
if (p.x <= midX) {
    
pointsL.add(p);
    
xL.add(p);
    
}
    
else {
    
pointsR.add(p);
    
xR.add(p);
    
}
    
}
    
for(Point p : Y) {
    
if (p.x <= midX) {
    
yL.add(p);
    
}
    
else {
    
yR.add(p);
    
}
    
}
    

    
// Conquer Part
    
Point[] closestPointsL = findClosestPair(pointsL, xL, yL);
    
Point[] closestPointsR = findClosestPair(pointsR, xR, yR);
    
double distL = Math.sqrt(Math.pow((closestPointsL[0].x - closestPointsL[1].x),2) +
    
Math.pow((closestPointsL[0].y - closestPointsL[1].y),2));
    
double distR = Math.sqrt(Math.pow((closestPointsR[0].x - closestPointsR[1].x),2) +
    
Math.pow((closestPointsR[0].y - closestPointsR[1].y),2));
    
Double min = distL < distR ? distL : distR;
    
if (distL < distR) {
    
closestPoints = closestPointsL;
    
} else {
    
closestPoints = closestPointsR;
    
}
    

    
// Combine part
    
List<Point> pointsDmin = new ArrayList<Point>();
    
for(Point p : Y) {
    
if ((p.x >= midX - min.intValue()) && (p.x <= midX + min.intValue())) {
    
pointsDmin.add(p);
    
}
    
}
    
for(Point p : pointsDmin) {
    
// check the next seven points that follow p for their distance from p as compared to min
    
for(int i=pointsDmin.indexOf(p); i < pointsDmin.indexOf(p)+7 && i < pointsDmin.size(); i++) {
    
double dist = Math.sqrt(Math.pow((p.x - pointsDmin.x),2) +
    
Math.pow((p.y - pointsDmin    .y),2));
    
if (dist < min) {
    
closestPoints[0] = p;
    
closestPoints[1] = pointsDmin    ;
    
break;
    
}
    
}
    
}
    
} else {
    
// use brute force method to find closest pair for 3 or less points
    
}
    

    
return closestPoints;
    
}

    Maintain a hashtable where keys are server IP addresses and values are connection requests. So, using Java syntax we have something like this:

    Map<String, List<ConnectionInfo>>

    Here server IP address is represented by a String. ConnectionInfo contains fields like timestamp, requester’s IP address. ConnectionInfo implements Comparable and is sorted on basis of timestamp in descending order.

    To obtain the number of connections in last 1 minute, do a binary search on the list for the first timestamp which was more than a minute ago.

    Time Complexity for this algorithm = O(log N), since key lookup in hashtable is roughly O(1).

    This seems to me “subset sum problem” in disguise, which is an NP-complete problem and has an exponential time complexity (http://en.wikipedia.org/wiki/Subset_sum_problem).

    The first observation here is that individual averages of the two partitions is the same as the average for the whole array.

    Let S=(x1,x2,…,xN) be the array of numbers

    Let m = (x1+x2+…+xN)/N be the average for whole array

    Let S1 and S2 be the partitions with u be the average for both of them. Then,

    u*#(S1) + u*#(S2) = sum of all numbers of array S = mN … #() means cardinality

    Since #(S1) + #(S2) = N, we have u=m.

    Now, the problem reduces basically to looking for a subset of S whose sum is a multiple of m i.e. km where k=1 to N.

    So, for each multiple km, you try to find a subset. As a result, worst-case complexity comes out to be a really horrible O(N*(2^N)).

    Let me know if you guys can glean on some missed detail and come up with a faster method.

    How about just doing inorder traversals of both the trees A and B? Then, you have basically two strings (lets assume for now that nodes contain single characters) and all you have to do is to check whether string for tree B is a substring of string for tree A.
