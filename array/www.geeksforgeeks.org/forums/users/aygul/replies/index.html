    If I got that right, no need for a DP solution with the whole solution space.
    Here is a O(n) c# implementation…

    [sourcecode language="C"]
    public static int FindLongestArithmeticProgression(int[] arr, out int s, out int e)
    {
    s = -1; e = -1;
    if (arr.Length == 0) return 0;

    if (arr.Length == 1) return 0;

    s = 0; e = 0;
    int curDiff = int.MinValue;
    int diff = curDiff;
    int tmp;

    int curLen = 0;
    int maxLen = 0;

    for (int i = 1; i < arr.Length; i++)
    {
    tmp = arr[i] – arr[i – 1];
    if (tmp != curDiff)
    {
    if (maxLen < curLen && curDiff > 0)
    {
    maxLen = curLen;
    e = i – 1;
    diff = curDiff;
    }
    curLen = 1;
    curDiff = tmp;
    }
    else
    {
    curLen++;
    }
    }

    s = e – maxLen;
    return diff;//output is valid if diff is > 0
    }

    [/sourcecode]

    @vishnuvardhan.2005

    The solution you’ve gave is O(n^2). Do the math… The second iteration will run from j=1 to n-1 which is n-1 times for the first time. Then from 2 to n-1, that’s n-2 times for the second one. And so on until j= n-1 which is 1 time. Total number of iterations in the inner loop : 1 + 2 + … + n-2 + n-1 (i guess the rest is straight-forward, but lets go on) that is (n-1)*n/2 = O(n^2).
