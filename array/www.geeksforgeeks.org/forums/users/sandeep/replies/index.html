  • June 10, 2011 at 12:56 PM in reply to: What is the time complexity #107866

    Sandeep
    Keymaster

    It can be proved using substitution method given in CLRS book.

    Time Complexity T(n) can be written as.

    T(n) = T(n-1) + T(n-2) + … T(1) + T(0).

    We can prove it using substitution method. Let the time complexity be O(2^(n-1))

    So T(n) <= c*2^(n-1) is true where c is a constant

    Let T(0) be d where d is another constant.

    T(n) = T(n-1) + T(n-2) + ... T(1) + T(0).
    
= c(2^(n-2) + 2^(n-3) + .. + 2^0) + d
    
= c(2^(n-1)) - C + d
    
<= c(2^(n-1)) as long as c > d

    Sandeep

    Use the Floyd’s Cycle-Finding Algorithm discussed @ http://geeksforgeeks.org/?p=112.


    
int detectloop(struct node *list)
    
{
    
struct node  *slow_p = list, *fast_p = list;
    

    
while(slow_p && fast_p &&
    
slow_p->next &&
    
fast_p->next &&
    
fast_p->next->next )
    
{
    
slow_p = slow_p->next;
    
fast_p  = fast_p->next->next;
    
if (slow_p == fast_p)
    
{
    
printf("Found Loop");
    
return 1;
    
}
    
}
    
return 0;
    
}

    We know that the loop terminates when both slow_p and fast_p meet meet. We also know that they will meet at a node which is a part of loop.

    After the above loop is over, fix slow_p to its current position. Now start from head and check every node whether it is part of loop or not. We can check this by iterating through the loop using slow_p. This is how we can get the pointer to first node in cycle (‘e’ in your example). Once we have it, it is easy to remove cycle.

  • May 3, 2011 at 4:31 AM in reply to: Amazon Interview Question for Software Engineer/Developer about Arrays #106757

    Sandeep
    Keymaster

    1) Binary search for the first occurrence of x in arr[]. Let the index of first occurrence be i.

    2) Binary search for the last occurrence of x in arr[]. Let the index of first occurrence be j.

    3) Return j-i+1;


    
/*Returns index of  the first occurrence of  x in arr[] of size n*/
    
int  first(int arr[], int low, int high, int x, int n)
    
{
    
if(high >= low)
    
{
    
int mid = (low + high)/2;  /*low + (high - low)/2;*/
    
if(( mid == 0 || x != arr[mid-1]) && (arr[mid] == x))
    
return mid;
    
else if(x > arr[mid])
    
return first(arr, (mid + 1), high, x, n);
    
else
    
return first(arr, low, (mid -1), x, n);
    
}
    
return -1;
    
}
    

    
/*Returns index of  the last occurrence of  x in arr[] of size n*/
    
int  last(int arr[], int low, int high, int x, int n)
    
{
    
if(high >= low)
    
{
    
int mid = (low + high)/2;  /*low + (high - low)/2;*/
    

    
if(( mid == n-1 || x  !=  arr[mid+1]) && (arr[mid] == x))
    
return mid;
    
else if(x > arr[mid])
    
return last(arr, (mid + 1), high, x, n);
    
else
    
return last(arr, low, (mid -1), x, n);
    
}
    
return -1;
    
}

    @lapil: This approach might not work in all the cases. XORs might be same for some invalid input.

    Following is a O(n) approach

    // n is the number of elements in arr[]

    bool checkConsecutive(arr[], n)

    ..1) Find the minimum element say min.

    ..2) Find the maximum element say max.

    ..3) If max – min + 1 is not equal to number elements in array, then return false.

    ..4) Create a temporary array count[] of size n.

    ..5) Initialize all the elements of count array as 0.

    ..6) Now iterate through arr[] and set the value of count using following formula.

    count[arr – min] = count[arr-min] + 1

    ..7) Scan the count array. If all values are 1 then return true, else return false.

    Time Complexity: O(n)

    Extra Space: O(n)
