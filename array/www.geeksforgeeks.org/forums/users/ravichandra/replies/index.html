    But how can this be an O(n) solution?

    This algorithm works like this.

    start k with index ‘0’ keep updating the sum(ak-bk) until it’s negative.

    If it is negative before k reaches it’s previous value then we will start with index 1, and repeat the same procedure until for some k, sum(ak-bk) is not negative.

    So this is going to be an O(N^2) solution.

    As they need O(n) solution and no space restrictions mentioned, better approach would be to use some extra space and use dynamic programming approach for this.

import java.util.Collection;
    
import java.util.HashMap;
    
import java.util.Iterator;
    
import java.util.StringTokenizer;
    
public class MainClass {
    
public static void main(String[] args)
    
{
    
frequency ("abc abc cde abc cde efg abc");
    

    
}
    
public static void frequency(String str)
    
{
    
StringTokenizer strTok = new StringTokenizer(str," ");
    
HashMap freqMap = new HashMap();
    
while (strTok.hasMoreTokens())
    
{
    
String token = strTok.nextToken();
    
Integer count = (Integer)freqMap.get(token);
    
if( count == null )
    
{
    
freqMap.put(token, new Integer(1));
    
}
    
else
    
{
    
int cnt = count.intValue();
    
cnt++;
    
freqMap.put(token, new Integer(cnt));
    
}
    
}
    
Collection c = freqMap.keySet();
    
Iterator it = c.iterator();
    
while(it.hasNext())
    
{
    
Object ob = it.next();
    
System.out.println( ob + " " + freqMap.get(ob));
    
}
    
}
    
}

    Assuming that the linked list has more than two nodes… the following code is a very simple way of achieving what you have asked. It is simply a swap operation.


    
void reverseFirstTwoNodes(struct node *head)
    
{
    
int temp;
    
temp = head->data;
    
head ->data = head->next->data;
    
head->next->data = temp;
    
}
