  • March 7, 2013 at 3:44 AM in reply to: Fill the array in sorted order. #115453

    rohitnsit08
    Participant

    line of thought :

    the question is based on finding the minimum element out of n elements repeatedly .  We can find minimum number in O(n) time by just scanning all the elements and finding the least number . here we are doing some computation which is going to be repeated after every new addition . that is where we can optimize the answer .

    Given we have already done comparison , is it necessary to do it again for every new element ?

    NO we can use Min-Heap

    important property of min-heap : heapifying one elements take log(n)  time but building the heap is expensive as it takes O(nlogn) time no actually it takes only O(n) time ( the O(nlogn) complexity which we normally find is light bound , tighter bound is O(n)  http://stackoverflow.com/questions/9755721/build-heap-complexity)

    so now it is easy .. just  make a min-heap and insert the root element in the new array { also store the index of the element in the root as we need to get the next element from the same array }

    insert the next element and call heapify which will again bring the lowest element on top in O(logn) time.

     

    total complexity will be m*n * O(logn) + O(n) = O(mnlogn)
