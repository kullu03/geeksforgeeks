    Simple solution:
    If frog is currently at rock i, then there are 2 possibilities, either he go to (i+1)th rock or (i+2)th rock.We can transform this into binary tree as follows. Start from rock 1 as root of tree. Every node can have at most two children.
    leftChild(root)=root+1
    rightChild(root)=root+2
    Terminate when u reach leaf node.
    Leaf node will be n.

    Now this is equivalent to print all root to leaf paths in binary tree.
    Code :

    void func(int arr[],int root,int ind,int n){
    if(root>n) return;
    arr[ind]=root;
    if(root==n){
        for(int i=0;i<=ind;i++)
            printf("%d ",arr[i]);
        printf("\n");
        return;
    }
    func(arr,root+1,ind+1,n);   // left child
    func(arr,root+2,ind+1,n);   // right child
}
void printPath(int n){
    int *arr=new int[n+1];
    func(arr,1,0,n);
    delete arr;
}

    For undirected Graph:
    It can be done in O(n) time preprocessing and O(n) extra space.
    Apply DFS/BFS and find out connected components as well as store parent and level of every vertex.
    For query1: If u and v are in same connected component then return 1, else return 0;// just check component number of u and v, O(1)

    For query2: find out LCA of u and v, lets say it is X.
    Now print path u->X->v.
    LCA can be found in O(l) becoz we have level and parent of each node, below is code for it:
    int LCA(int u,int v){
    if(u==v) return u;
    if(level[u]>level[v]) u=parent[u];
    else if(level[u] else {
    u=parent[u]; v=parent[v];
    }
    }
    // parent and level for each node can be calculated using BFS/DFS, which corresponds to BFS/DFS tree.

    A better approach: O(log(n^2))
    nth Fibonacci can be calculated in log(n) time by using matrix exponentiation,have a look at

    http://www.geeksforgeeks.org/program-for-nth-fibonacci-number/

    Now apply binary search which will run in log(n) time.
    int high=1,low=0;
    while(fibo(high) low=high;
    high*=2;
    }
    // where fibo(i) calculates ith fibonacci number in log(i) time
    int answer=binarySearch(n,low,high);
    // int binarySearch(int n,int low,int high); // now n lies : fibo(low) As fibonacci numbers are in increasing order, we can apply binary search.

    Complexity :
    Binary Search : log(n)
    At each step of binary search we have to call fibo((low+high)/2), which is O(log(n)) operation,
    so overall complexy is log(n^2), which is better than brute force O(n) approach for larger n.
