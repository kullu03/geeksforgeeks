    Let x= # 3 paise stamps and y=# 5 paise stamps

    Then 3x + 5y =100 paise

    Now x and y can take only integer values.

    Particularly y can take one of {0,1,2,…..,20}.

    Since 100-5y must be multiple of 5, ( 100 – 5y = 5(20 – y) ),

    3x is multiple of 5 as well as 3.

    So, 3x is multiple of 15.

    =>

    Following are valid pairs of 3x + 5y =100

    (15, 85), (30, 70), (45, 55 ), (60, 40), (75, 25), (90, 10)

    Corresponding valid (x,y) pairs are

    (5, 17), (10, 14), (15, 11), (20, , (25, 5), (30, 2)

    So valid total number of stamps

    22, 24, 26, 28, 30, 32

    Anything else is invalid.

    Approach by Dedicated Programmer takes time

    1) nlogn for sorting array.

    2) mlogn for binary searching each element of second array in first sorted.

    Total = O(m+n)logn

    Approach by Diksha takes

    1) nlogn and mlogm for sorting both arrays.

    2) m+n for running two pointers.

    m+n+mlogm+nlogn = O(mlogm+nlogn)

    If m and n are comparable both are almost same, but if n<<m then algorithm by Dedicated Programmer takes less time. Since we can always sort only smaller array.

    Got a vague idea.

    The rows and columns are sorted let us say in increasing order. By this do we mean each row and column are increasing sequences?

    If so, consider following array

    A11 A12 A13 A14

    A21 A22 A23 A24

    A31 A32 A33 A34

    Then consider a structure like this (We are not building it, just for illustration:)

    A11

    A12………..A21

    A13 A22…..A22 A31

    A14 A23…..A23 A32

    A33………..A24

    A44

    Now consider matrix:

    ………..1……….. 5…………… 6………………. 7

    ………..2 ……….7……………. 8………………. 9

    ………..3………. 8……………. 9……………. 10

    Sorted sequence will be ->

    1 2 3 5 6 7 8 9 10

    Ok, now the idea is start two travellers starting at the arr[0][0]. Of course if k=1, arr[0][0] is ans.

    Now Row_Traveller travels across rows (downwords) and Column_Traveller travels across columns (rightwords). Whoever sees a smaller value travels one position ahead in its travelling direction.

    Meanwhile we keep a counter of total no of positions travelled by both. Once it reaches k we have our answer at the latest location skipped by either of traveller.

    Also, when row traveller reaches last row, it advances to next column and hence column traveller also advances. Be careful at such boundary cases.

    If both elements are equal both will advance and count increaments by 1 (If we want same elements to be counted only once) .

    Example:

    Col_Traveller —>

    Row_Traveller

    |

    V

    ………..1……….. 5…………… 6………………. 7

    ………..2 ……….7……………. 8………………. 9

    ………..3………. 8……………. 9……………. 10

    Let k = 6

    Then travelling is as follows:

    Row_Traveller…………….Col_Traveller………………..Count(Elements skipped)

    2,1………………………………….1,2…………………………………….1

    3,1………………………………….1,2…………………………………….2

    1,2………………………………….1,3…………………………………….3

    2,2………………………………….1,3…………………………………….4

    2,2………………………………….1,4…………………………………….5

    3,2………………………………….2,3…………………………………….6

    Since count reaches k, the latest element skipped that is arr[2][2] = 7 is the answer.

    Please note how we advance the Col_Traveller.

    I think complexity is O(M x N).

    Any suggestions or modifications?
