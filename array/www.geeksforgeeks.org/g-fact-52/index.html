G-Fact 13

const Behaviour in C and C++

In C, the const qualified identifiers will have external linkage, where as in C++ it will have internal linkage. For example,

In C++, the following statement

float const interest_rate = 9.25;

is implicitly defined as

static float const interest_rate = 9.25;

i.e. the scope of interest_rate is limited to the block in which it is defined.

In C, the above statement will have external linkage when defined at file scope, i.e. it will be visible outside the current translation unit (source file).

The internal linkage of const qualified variables have some advantages in C++. We will cover them in next article.

      • Sreenivas Doosa

        The result is as expected. The reason here is.

        When you declare a variable as constant, that will be stored in symbol table. At this point in your example there is no address is allocated for variable ‘c’ and when you print ‘c’, the value is taken from symbol table. But when you explicitly tries to access the address of the constant variable ‘c’ then the compiler will allocate an address for ‘c’ and this is different from the one stored in symbol table.
        Now the ptr refers to the address of ‘c’, so when you change the value at ptr address, the value at the address of ‘c’ is changed but it will not be changed in symbol table.

        Finally when you print just ‘c’, the value fetched from symbol table.
        If you print *ptr, the value is fetched from address of ‘c’.
