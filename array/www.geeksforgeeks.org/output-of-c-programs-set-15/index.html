Output of C Programs | Set 15

Predict the output of below C programs.

Question 1

#include<stdio.h>
int main(void)
{
  int a = 1;
  int b = 0;
  b = ++a + ++a;
  printf("%d %d",a,b);
  getchar();
  return 0;
}

Output: Undefined Behavior
See http://en.wikipedia.org/wiki/C_syntax#Undefined_behavior )



Question 2

#include<stdio.h>

int main()
{
  int a[] = {1, 2, 3, 4, 5, 6};
  int *ptr = (int*)(&a+1);
  printf("%d ", *(ptr-1) );
  getchar();
  return 0;
}

Output: 6
&a is address of the whole array a[]. If we add 1 to &a, we get “base address of a[] + sizeof(a)”. And this value is typecasted to int *. So ptr – 1 points to last element of a[]

    Question 1:
    It is good point to note. There is a concept called ‘sequence point’ in C and C++, ‘a point is program execution where the dust due to previous expression will be settled down’. In the example, there is no sequence point in ‘+’ operation. So, the value of ‘a’ is not flushed to ‘a’ location till the operation is performed. Also, the order of operands evaluation is not guaranteed (depends on the order of pushing them on to stack). A more detailed discussion can be found on the following links,

    http://en.wikipedia.org/wiki/Sequence_point
    http://msdn.microsoft.com/en-us/library/azk8zbxd(VS.80).aspx
    http://msdn.microsoft.com/en-us/library/d45c7a5d(VS.80).aspx

    Question 2:
    It is worth noting the emphasis on type information of program elements in C/C++. The array ‘a’ is of type ‘array of 6 (N) integers’. When we apply & (in C) operator on that array, it will be considered as ‘constant pointer to array of 6 integers’. Obviously, an addition operation depends on the type of operand (array of N integers) which is ‘constant pointer to array of 6 integers’. Thus, the expression ‘(&a+1)’ yields “base address of a[] + sizeof(a)”. The same effect can be obtained as below,

     

#include<stdio.h>
 
int main()
{
  int array[] = {1, 2, 3, 4, 5, 6};
  int (*ptr)[6] = &array;
  
  printf("%d ", *((int *)(ptr + 1) - 1));
 
  return 0;
}
