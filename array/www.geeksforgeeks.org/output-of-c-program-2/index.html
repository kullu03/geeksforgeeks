Output of C++ Program | Set 17

Predict the output of following C++ programs.

Question 1

#include <iostream>
using namespace std;

class A
{
    public:
    A& operator=(const A&a)
    {
        cout << "A's assignment operator called" << endl;
        return *this;
    }
};

class B
{
    A a[2];
};

int main()
{
    B b1, b2;
    b1 = b2;
    return 0;
}

Output: 

A's assignment operator called
A's assignment operator called

The class B doesn’t have user defined assignment operator. If we don’t write our own assignment operator, compiler creates a default assignment operator. The default assignment operator one by one copies all members of right side object to left side object. The class B has 2 members of class A. They both are copied in statement “b1 = b2″, that is why there are two assignment operator calls.



Question 2

#include<stdlib.h>
#include<iostream>

using namespace std;

class Test {
public:
    void* operator new(size_t size);
    void operator delete(void*);
    Test() { cout<<"\n Constructor called"; }
    ~Test() { cout<<"\n Destructor called"; }
};

void* Test::operator new(size_t size)
{
    cout<<"\n new called";
    void *storage = malloc(size);
    return storage;
}

void Test::operator delete(void *p )
{
    cout<<"\n delete called";
    free(p);
}

int main()
{
    Test *m = new Test();
    delete m;
    return 0;
}

 new called
 Constructor called
 Destructor called
 delete called

Let us see what happens when below statement is executed. 

    Test *x = new Test;

When we use new keyword to dynamically allocate memory, two things happen: memory allocation and constructor call. The memory allocation happens with the help of operator new. In the above program, there is a user defined operator new, so first user defined operator new is called, then constructor is called.
The process of destruction is opposite. First, destructor is called, then memory is deallocated.

    We can overload the operator new and similarly delete and its array version delete[].

    The overloaded new operator receives a parameter of type size_t by default from compiler, which is the number of bytes of memory to be allocated. The compiler calculates the total size of the member variables of the class and provides it to the overloaded new

    In overloaded new we allocate memory by malloc. After our overloaded new returns, the compiler automatically calls the suitable constructor.

     
#include<iostream>
#include<stdlib.h>
using namespace std;

class MyClass{
  int m_i;
  float m_f;
  public:
    MyClass(){
      m_i=0;
      m_f=0;
      cout << "\nIn default constructor(m_i, m_f) =("<<m_i<<", " <<m_f<<")\n";
    }
    MyClass(int i, float f){
      m_i=i;
      m_f=f;
      cout << "\nIn parametarized constructor(m_i, m_f) =("<<m_i<<", " <<m_f<<")\n";
    }

    ~MyClass(){
       cout << "\nIn Destructor";
    }
    void operator delete(void* ptr) {
       free(ptr);
    }
    void operator delete[](void* ptr) {
       free(ptr);
    }
    void show() {
       cout << "In show: m_i = "<< m_i << ", m_f =" << m_f<< endl;
    }
};

// global definition of new
void* operator new(size_t size) {
       void* ptr= malloc(size);
       cout << "\noperator new(size_t size): size  " << size;
       return ptr;
}

int main() {
  cout << "------obj_ptr= new MyClass(5, 1.2)---------\n";
  MyClass* obj_ptr= new MyClass(5, 1.2);
  obj_ptr->show();
  delete obj_ptr ;

  cout << "------obj_ptr1= new MyClass[1]---------\n";
  MyClass* obj_ptr1= new MyClass[2];
  obj_ptr1->show();
  (obj_ptr1+1)->show();
  delete[] obj_ptr1;

  return 0;
}

    Why overload new and delete?
    1. To take charge or control over how to allocate memory
    2. To aid in debugging; keep track of memory allocation and deallocation in the program
    3. To do some other operation apart from allocating memory at the time of memory allocation/deallocation
    Note: The return type of the overloaded new must be void*. The overloaded delete operator receives a parameter of type void*.

    Why compiler given size is printed with extra 8-byte in overloaded new, when destructor is defined ? If destructor is not defined then for the above case obj_ptr1= new MyClass[2], size in overloaded new by the compiler is 16 ie 2*8.(In 32 bit)

    Want to add something more. There is a difference between the two things
    1. new operator
    2. operator new

    “operator new” just allocates the number of bytes it is given and it returns the raw pointer to that memory allocated.

     
void *memory = operator new(sizeof(Test));

    This is defined in language standards. We can overload this operator new according to our needs.

    secondly “new operator” calls the “DEFAULT” constructor on the memory allocated by operator new, note that it only calls default constructor.

     
call Test::Test()    // initialize the
on *memory;          // object on memory
 
     
Test *t = static_cast<Test *>(memory)

    Now “t: is pointed to new obect.

    Thus whenever we call “new operator” .. it first calls “operator new” which returns the pointer of memory allocated and then “new operator” calls default constructor.

      The term “new” mean a keyword in main() and the operator function that takes a size as parameter is “new operator”.

      When we write a “new” to allocate memory, compiler does following things:

      1) The compiler calculates the size of memory to be allocated.

      2) Calls “new operator” with size as a parameter.

      3) Once the call to “new operator” is over, calls constructor.
